# Loading States and Suspense - Teaching Notes

## Topic 21: Streaming, Skeletons, and Progressive Page Rendering

---

# 1. Why This Topic Matters

Users abandon slow pages. Loading states tell users the app is working — not broken. Next.js App Router supports **streaming**: send HTML to the browser progressively as each section becomes ready.

Without loading UI, users stare at a blank screen during data fetches. With `loading.tsx` and Suspense boundaries, they see the page shell and skeleton placeholders immediately. This topic covers automatic loading files, manual Suspense, and skeleton patterns.

---

# 2. How Streaming Works in Next.js

```text
Browser requests /dashboard
        │
        ▼
Server starts rendering
        │
        ├──► Fast content (header, nav) ──► sent immediately
        │
        ├──► Slow section A (still loading) ──► fallback HTML sent
        │         └── data ready ──► streamed in
        │
        └──► Slow section B ──► fallback HTML sent
                  └── data ready ──► streamed in
```

| Without streaming | With streaming |
| ----------------- | -------------- |
| Wait for ALL data | Send fast parts immediately |
| Blank screen for 3s | Shell visible in ~100ms |
| All-or-nothing render | Progressive section-by-section |

# 3. loading.tsx — Automatic Loading UI

Place `loading.tsx` in any route folder. Next.js wraps the page in a Suspense boundary automatically.

```text
app/
  dashboard/
    loading.tsx     ← shown while dashboard/page.tsx loads
    page.tsx
  products/
    loading.tsx
    page.tsx
    [id]/
      loading.tsx   ← shown while product detail loads
      page.tsx
```

```tsx
// app/products/loading.tsx
export default function ProductsLoading() {
  return (
    <main aria-busy="true" aria-label="Loading products">
      <h1>Products</h1>
      <div className="grid grid-cols-3 gap-4">
        {Array.from({ length: 6 }).map((_, i) => (
          <div
            key={i}
            className="h-48 animate-pulse rounded-lg bg-gray-200"
            aria-hidden="true"
          />
        ))}
      </div>
    </main>
  )
}
```

| Feature | Detail |
| ------- | ------ |
| Instant | Shows immediately on navigation |
| Scoped | Only covers the route segment it's in |
| Preserves layout | Parent `layout.tsx` renders immediately |
| No `'use client'` needed | Can be a Server Component |

# 4. React Suspense Boundaries

Use `<Suspense>` manually for finer control within a page.

```tsx
// app/dashboard/page.tsx
import { Suspense } from 'react'
import { StatsCards } from '@/components/stats-cards'
import { RevenueChart } from '@/components/revenue-chart'
import { RecentOrders } from '@/components/recent-orders'
import { StatsSkeleton } from '@/components/stats-skeleton'
import { ChartSkeleton } from '@/components/chart-skeleton'
import { OrdersSkeleton } from '@/components/orders-skeleton'

export default function DashboardPage() {
  return (
    <main>
      <h1>Dashboard</h1>

      {/* Each section streams independently */}
      <Suspense fallback={<StatsSkeleton />}>
        <StatsCards />
      </Suspense>

      <Suspense fallback={<ChartSkeleton />}>
        <RevenueChart />
      </Suspense>

      <Suspense fallback={<OrdersSkeleton />}>
        <RecentOrders />
      </Suspense>
    </main>
  )
}
```

```tsx
// components/stats-cards.tsx — async Server Component
async function fetchStats() {
  const res = await fetch('https://api.example.com/stats', { cache: 'no-store' })
  return res.json()
}

export async function StatsCards() {
  const stats = await fetchStats()

  return (
    <div className="grid grid-cols-4 gap-4">
      <div className="rounded-lg border p-4">
        <p className="text-sm text-gray-500">Total Users</p>
        <p className="text-2xl font-bold">{stats.totalUsers}</p>
      </div>
      <div className="rounded-lg border p-4">
        <p className="text-sm text-gray-500">Revenue</p>
        <p className="text-2xl font-bold">Rs. {stats.revenue}</p>
      </div>
    </div>
  )
}
```

| Approach | Granularity |
| -------- | ----------- |
| `loading.tsx` | Entire route segment |
| `<Suspense>` in page | Individual sections within a page |
| Both together | Route-level + section-level loading |

# 5. Skeleton UI Patterns

Skeletons mimic the shape of real content — better UX than spinners.

```tsx
// components/product-card-skeleton.tsx
export function ProductCardSkeleton() {
  return (
    <div className="animate-pulse rounded-lg border p-4" aria-hidden="true">
      <div className="mb-4 h-40 rounded bg-gray-200" />
      <div className="mb-2 h-4 w-3/4 rounded bg-gray-200" />
      <div className="mb-4 h-4 w-1/2 rounded bg-gray-200" />
      <div className="h-8 w-full rounded bg-gray-200" />
    </div>
  )
}
```

```tsx
// components/products-grid-skeleton.tsx
import { ProductCardSkeleton } from './product-card-skeleton'

export function ProductsGridSkeleton({ count = 6 }: { count?: number }) {
  return (
    <div
      className="grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3"
      aria-busy="true"
      aria-label="Loading products"
    >
      {Array.from({ length: count }).map((_, i) => (
        <ProductCardSkeleton key={i} />
      ))}
    </div>
  )
}
```

| Pattern | Best for |
| ------- | -------- |
| Pulse animation | Cards, lists, text blocks |
| Fixed dimensions | Prevent layout shift (CLS) |
| `aria-busy="true"` | Screen reader accessibility |
| Match real layout | Skeleton should mirror final UI shape |

# 6. Avoiding Layout Shift (CLS)

Core Web Vitals penalize content that jumps when data loads. Reserve space in skeletons.

```tsx
// ❌ Bad — skeleton smaller than real content, causes jump
<div className="h-20 animate-pulse bg-gray-200" />

// ✅ Good — skeleton matches real card dimensions
<div className="h-64 w-full animate-pulse rounded-lg bg-gray-200" />
```

```tsx
// components/user-avatar-skeleton.tsx
export function UserAvatarSkeleton() {
  return (
    // Fixed size matches real avatar component
    <div className="h-10 w-10 animate-pulse rounded-full bg-gray-200" aria-hidden="true" />
  )
}
```

| Technique | CLS impact |
| --------- | ---------- |
| Fixed height skeletons | ✅ Low shift |
| `aspect-ratio` on image skeletons | ✅ Low shift |
| Spinner with no reserved space | ❌ High shift |
| Content appearing above existing content | ❌ High shift |

# 7. Nested Suspense Boundaries

Nest Suspense for complex pages with independent slow sections.

```tsx
// app/analytics/page.tsx
import { Suspense } from 'react'

export default function AnalyticsPage() {
  return (
    <main>
      <h1>Analytics</h1>

      {/* Outer: entire overview section */}
      <Suspense fallback={<p>Loading overview...</p>}>
        <OverviewSection />
      </Suspense>

      <div className="grid grid-cols-2 gap-6">
        {/* Inner: each chart independent */}
        <Suspense fallback={<ChartSkeleton />}>
          <SalesChart />
        </Suspense>

        <Suspense fallback={<ChartSkeleton />}>
          <TrafficChart />
        </Suspense>
      </div>
    </main>
  )
}
```

```text
Analytics Page Render Timeline:

0ms   ──► Header + "Analytics" title sent
100ms ──► Overview skeleton sent
500ms ──► Overview data streamed in
100ms ──► Chart skeletons sent (both)
800ms ──► Sales chart streamed in
1200ms ──► Traffic chart streamed in
```

# 8. Loading States for Client Components

Client components using `useSearchParams` or TanStack Query need their own loading patterns.

```tsx
'use client'

import { Suspense } from 'react'
import { useSearchParams } from 'next/navigation'

function FilterResultsInner() {
  const searchParams = useSearchParams()
  const category = searchParams.get('category')
  // fetch and render filtered results...
  return <p>Showing: {category ?? 'All'}</p>
}

export function FilterResults() {
  return (
    <Suspense fallback={<ProductsGridSkeleton count={4} />}>
      <FilterResultsInner />
    </Suspense>
  )
}
```

```tsx
'use client'

import { useQuery } from '@tanstack/react-query'

export function LiveNotifications() {
  const { data, isLoading, isError } = useQuery({
    queryKey: ['notifications'],
    queryFn: fetchNotifications,
  })

  if (isLoading) return <NotificationsSkeleton />
  if (isError) return <p role="alert">Could not load notifications</p>

  return (
    <ul>
      {data?.map((n) => (
        <li key={n.id}>{n.message}</li>
      ))}
    </ul>
  )
}
```

| Context | Loading approach |
| ------- | ---------------- |
| Server Component | `loading.tsx` or `<Suspense>` |
| Client with `useSearchParams` | Wrap in `<Suspense>` |
| TanStack Query | `isLoading` state + skeleton |
| Server Action pending | `useActionState` + `isPending` |

# 9. loading.tsx vs Suspense — When to Use Which

| Scenario | Use |
| -------- | --- |
| Entire page loads slowly | `loading.tsx` in route folder |
| Page shell is fast, one section slow | `<Suspense>` around slow component |
| Multiple independent slow sections | Multiple `<Suspense>` boundaries |
| Navigation between routes | `loading.tsx` (automatic on route change) |
| Client component with async deps | `<Suspense>` wrapper |

```tsx
// app/products/page.tsx — combine both
import { Suspense } from 'react'
import { ProductFilters } from '@/components/product-filters'
import { ProductGrid } from '@/components/product-grid'
import { ProductsGridSkeleton } from '@/components/products-grid-skeleton'

// loading.tsx handles initial route navigation
// Suspense handles slow product grid within the page

export default function ProductsPage() {
  return (
    <main>
      <h1>Products</h1>
      <ProductFilters /> {/* fast — renders immediately */}

      <Suspense fallback={<ProductsGridSkeleton />}>
        <ProductGrid /> {/* slow — streams in */}
      </Suspense>
    </main>
  )
}
```

# 10. Prefetching and Loading Perception

`Link` prefetches routes in the background — loading.tsx may flash briefly or not at all.

```tsx
import Link from 'next/link'

// Default — prefetches when link enters viewport
<Link href="/products">Products</Link>

// Disable prefetch for rarely visited pages
<Link href="/admin/settings" prefetch={false}>Settings</Link>
```

| Technique | Perceived speed |
| --------- | --------------- |
| `Link` prefetch | Route ready before click — instant navigation |
| Skeleton UI | User sees progress immediately |
| Streaming | Shell visible while data loads |
| Optimistic UI | Instant feedback before server confirms |

# 11. Best Practices

✔ Always provide loading UI — never leave users on a blank screen

✔ Use skeleton components that match the shape of real content

✔ Set fixed dimensions on skeletons to prevent layout shift (CLS)

✔ Split slow data fetches into separate async Server Components with Suspense

✔ Use `loading.tsx` for route-level loading on navigation

✔ Wrap `useSearchParams` components in `<Suspense>`

✔ Add `aria-busy="true"` and `aria-label` to loading containers for accessibility

# 13. Summary

| Tool | Scope | Trigger |
| ---- | ----- | ------- |
| `loading.tsx` | Route segment | Navigation + async page render |
| `<Suspense fallback={...}>` | Any component subtree | Async child component |
| Skeleton components | Visual placeholder | Used inside fallbacks |
| Streaming | Server → browser | Progressive HTML delivery |
| `aria-busy` | Accessibility | Screen reader loading state |
| Fixed dimensions | CLS prevention | Skeleton matches real layout |
