# Environment Setup - Teaching Notes

## Topic 3: Setting Up a Next.js Development Environment

---

# 1. Why This Topic Matters

Students cannot learn Next.js without a working local environment. A correct setup means they can:

* Create Next.js projects with one command
* Run a dev server with Fast Refresh (hot reload)
* Navigate the `app/` folder structure confidently
* Install libraries with npm
* Use TypeScript and ESLint from day one
* Build and preview production output

This lesson covers **Node.js**, **npm**, **create-next-app**, project structure, npm scripts, and browser tooling — adapted from the React environment setup in react-project-study but for Next.js.

# 3. Install and Verify Node.js

Download the **LTS** (Long Term Support) version from [https://nodejs.org](https://nodejs.org).

Recommended: **Node.js 20.x LTS** or **22.x LTS** for Next.js 15.

After installation, verify in the terminal:

```bash
node -v
npm -v
```

Example output:

```text
v20.11.0
10.2.4
```

| Command | Meaning |
| ------- | ------- |
| `node -v` | Node runtime version |
| `npm -v` | npm version bundled with Node |

If commands are not found:

* Restart the terminal after install
* On Windows, ensure Node is in PATH
* On Mac/Linux, consider installing via `nvm` if you need multiple versions

# 4. What npm Does

**npm** (Node Package Manager) ships with Node. It:

* Installs packages from the npm registry into `node_modules`
* Reads `package.json` for project metadata and scripts
* Locks versions in `package-lock.json` for reproducible installs

### Common commands

```bash
npm install              # Install dependencies from package.json
npm install zod          # Add a package and update package.json
npm uninstall zod        # Remove a package
npm run dev              # Run the "dev" script
npm update               # Update packages within semver ranges
```

| File | Role |
| ---- | ---- |
| `package.json` | Project name, scripts, dependency list |
| `package-lock.json` | Exact installed versions (commit this file) |
| `node_modules/` | Downloaded packages (do not edit by hand; often gitignored) |

# 5. Create a New Next.js Project

Use **create-next-app** — the official scaffolding tool for Next.js.

```bash
npx create-next-app@latest my-next-app
```

Interactive prompts (recommended choices for this course):

```text
✔ Would you like to use TypeScript? … Yes
✔ Which linter would you like to use? … ESLint
✔ Would you like to use Tailwind CSS? … Yes
✔ Would you like your code inside a `src/` directory? … No (or Yes — both work)
✔ Would you like to use App Router? … Yes
✔ Would you like to use Turbopack for `next dev`? … Yes
✔ Would you like to customize the import alias (@/* by default)? … No
```

Then:

```bash
cd my-next-app
npm run dev
```

Terminal shows:

```text
  ▲ Next.js 15.x.x
  - Local:        http://localhost:3000
  - Turbopack (dev)

 ✓ Starting...
 ✓ Ready in 800ms
```

Open **http://localhost:3000** in the browser.

### Compare with React (Vite) setup

| Step | React (react-project-study) | Next.js |
| ---- | --------------------------- | ------- |
| Create command | `npm create vite@latest` | `npx create-next-app@latest` |
| Dev port | 5173 | 3000 |
| Dev script | `npm run dev` | `npm run dev` |
| Entry file | `src/main.tsx` | `app/layout.tsx` + `app/page.tsx` |

# 6. Default Next.js Project Folder Structure

```text
my-next-app/
├── app/
│   ├── favicon.ico
│   ├── globals.css          # Global styles
│   ├── layout.tsx           # Root layout (HTML shell)
│   └── page.tsx             # Home page (/)
├── public/
│   ├── next.svg
│   └── ...                  # Static assets served at /
├── node_modules/            # Installed packages (gitignored)
├── .gitignore
├── eslint.config.mjs        # ESLint configuration
├── next.config.ts           # Next.js configuration
├── next-env.d.ts            # Next.js TypeScript types (auto-generated)
├── package.json
├── package-lock.json
├── postcss.config.mjs       # PostCSS (for Tailwind)
├── README.md
└── tsconfig.json            # TypeScript configuration
```

If you chose the `src/` directory option:

```text
my-next-app/
├── src/
│   └── app/
│       ├── layout.tsx
│       └── page.tsx
├── public/
└── ...
```

| Folder / file | Purpose |
| ------------- | ------- |
| `app/` | App Router — pages, layouts, route handlers |
| `public/` | Static files (images, fonts) served at root URL |
| `components/` | Not created by default — add for shared UI |
| `next.config.ts` | Next.js settings (images, redirects, env) |
| `tsconfig.json` | TypeScript compiler options |
| `eslint.config.mjs` | Linting rules |

Compare with react-project-study:

```text
react-project-study/
├── src/
│   ├── main.tsx             # Entry — mounts React
│   ├── App.tsx              # Root component + routes
│   ├── pages/               # Page components (not file-based routing)
│   ├── components/          # Reusable UI
│   └── api/                 # API helpers
├── public/
├── index.html               # HTML shell (Vite injects JS here)
└── vite.config.ts
```

Next.js has **no `index.html`** and **no `main.tsx`** — the framework handles bootstrapping.

# 7. Key Files Explained

### `app/layout.tsx` — Root layout

Every App Router app must have a root layout with `<html>` and `<body>`:

```tsx
// app/layout.tsx
import type { Metadata } from 'next'
import './globals.css'

export const metadata: Metadata = {
  title: 'My Next App',
  description: 'Training project',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

This replaces the role of `index.html` + `main.tsx` in a Vite React app.

### `app/page.tsx` — Home page

```tsx
// app/page.tsx
export default function HomePage() {
  return (
    <main>
      <h1>Welcome to Next.js Training</h1>
      <p>Edit app/page.tsx to get started.</p>
    </main>
  )
}
```

Equivalent to the default content in `App.tsx` or `Home.tsx` in react-project-study.

### `next.config.ts` — Framework configuration

```typescript
// next.config.ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  /* config options here */
}

export default nextConfig
```

Common uses: image domains, redirects, environment-specific settings. Leave default for now.

# 8. package.json and npm Scripts

After create-next-app, `package.json` looks similar to this:

```json
{
  "name": "my-next-app",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  },
  "dependencies": {
    "next": "15.x.x",
    "react": "19.x.x",
    "react-dom": "19.x.x"
  },
  "devDependencies": {
    "@types/node": "^20",
    "@types/react": "^19",
    "@types/react-dom": "^19",
    "eslint": "^9",
    "eslint-config-next": "15.x.x",
    "typescript": "^5"
  }
}
```

| Script | Command | Purpose |
| ------ | ------- | ------- |
| `dev` | `npm run dev` | Start development server with Fast Refresh |
| `build` | `npm run build` | Create optimized production build |
| `start` | `npm start` | Run production server (after build) |
| `lint` | `npm run lint` | Check code with ESLint |

### Development workflow

```bash
# Terminal 1 — keep running while coding
npm run dev

# Terminal 2 — run commands as needed
npm run lint
npm run build
```

Compare with react-project-study scripts:

| Script | Vite React | Next.js |
| ------ | ---------- | ------- |
| Dev | `vite` → port 5173 | `next dev` → port 3000 |
| Build | `vite build` → `dist/` | `next build` → `.next/` |
| Preview | `vite preview` | `next start` |

# 9. TypeScript Setup in Next.js

create-next-app configures TypeScript automatically. Key files:

### `tsconfig.json`

```json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [{ "name": "next" }],
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

| Setting | Meaning |
| ------- | ------- |
| `"strict": true` | Enables strict type checking |
| `"paths": { "@/*": ["./*"] }` | `@/components/Button` imports from project root |
| `"jsx": "preserve"` | Next.js handles JSX transformation |

### Typing a page component

```tsx
// app/about/page.tsx
export default function AboutPage() {
  return <h1>About Us</h1>
}
```

Next.js infers types for pages. Explicit props typing comes in later topics (dynamic routes, search params).

# 10. ESLint Setup

Next.js ships with ESLint configured for App Router best practices.

```javascript
// eslint.config.mjs (simplified)
import { dirname } from 'path'
import { fileURLToPath } from 'url'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  baseDirectory: dirname(fileURLToPath(import.meta.url)),
})

const eslintConfig = [
  ...compat.extends('next/core-web-vitals', 'next/typescript'),
]

export default eslintConfig
```

Run linting:

```bash
npm run lint
```

Common ESLint rules enforced:

* No unused variables
* Correct hook usage in client components
* Next.js-specific rules (e.g. no `<img>` — use `next/image`)

Fix auto-fixable issues:

```bash
npm run lint -- --fix
```

# 13. Running the Dev Server

```bash
cd my-next-app
npm run dev
```

| URL | What you see |
| --- | ------------ |
| http://localhost:3000 | Home page (`app/page.tsx`) |
| Edit `app/page.tsx` | Page updates instantly (Fast Refresh) |
| Add `app/about/page.tsx` | New route at `/about` automatically |

### Fast Refresh vs full reload

| Change type | Behavior |
| ----------- | -------- |
| Edit component JSX | Fast Refresh — state preserved in client components |
| Edit Server Component | Server re-renders; page refreshes |
| Add new file in `app/` | New route available immediately |
| Edit `next.config.ts` | Requires server restart |

Stop the dev server with `Ctrl + C` in the terminal.

# 15. Common Setup Problems

| Problem | Likely fix |
| ------- | ---------- |
| `command not found: npx` | Install Node.js; restart terminal |
| Port 3000 in use | Stop other process or run `npm run dev -- -p 3001` |
| `Module not found` | Run `npm install` in project root |
| TypeScript errors in new file | Check file is `.tsx` for JSX; restart TS server in editor |
| Changes not visible | Hard refresh (Ctrl+Shift+R); check correct project folder |
| `next: command not found` | Run from folder containing `package.json` |
| Turbopack errors | Retry with `next dev` (without Turbopack flag) |

### Clear install (last resort)

```bash
rm -rf node_modules .next package-lock.json
npm install
```

# 16. Best Practices for Development Setup

✔ Use **Node LTS** for stability in classrooms and teams

✔ Commit **`package-lock.json`** so everyone gets the same versions

✔ Run projects from the **folder that contains `package.json`**

✔ Keep **one terminal** running `npm run dev` while coding

✔ Choose **TypeScript + ESLint + Tailwind** during create-next-app

✔ Use **App Router** — never Pages Router for new projects

✔ Name projects with **lowercase and hyphens** (`my-next-app`)

✔ Do not edit files inside `node_modules` or `.next`
