# Forms and User Input - Teaching Notes

## Topic 11: Controlled Inputs, Server Actions, and Validation

---

# 1. Why This Topic Matters

Forms are how users create, update, and delete data. Next.js supports both **traditional client-side forms** and **Server Actions** that work even without JavaScript.

Students must understand:

* Controlled inputs in Client Components
* The HTML `action` attribute with server functions
* Client validation for UX, server validation for security
* Submit loading states and post-submit revalidation

Poor form handling leads to security holes, lost input on errors, and confusing submit feedback.

# 2. Client-Side Forms with Controlled Inputs

Interactive forms with instant validation require **Client Components** with **`useState`**.

```tsx
// components/forms/ContactForm.tsx
'use client'

import { useState, FormEvent } from 'react'

interface FormData {
  name: string
  email: string
  message: string
}

interface FormErrors {
  name?: string
  email?: string
  message?: string
}

export function ContactForm() {
  const [formData, setFormData] = useState<FormData>({
    name: '',
    email: '',
    message: '',
  })
  const [errors, setErrors] = useState<FormErrors>({})

  function handleChange(field: keyof FormData, value: string) {
    setFormData((prev) => ({ ...prev, [field]: value }))
    // Clear error when user types
    if (errors[field]) {
      setErrors((prev) => ({ ...prev, [field]: undefined }))
    }
  }

  function validate(): FormErrors {
    const newErrors: FormErrors = {}
    if (!formData.name.trim()) newErrors.name = 'Name is required'
    if (!formData.email.includes('@')) newErrors.email = 'Valid email required'
    if (formData.message.length < 10) newErrors.message = 'Message too short'
    return newErrors
  }

  function handleSubmit(event: FormEvent) {
    event.preventDefault()
    const validationErrors = validate()
    if (Object.keys(validationErrors).length > 0) {
      setErrors(validationErrors)
      return
    }
    console.log('Submit:', formData)
    // later: call Server Action or API
  }

  return (
    <form onSubmit={handleSubmit} noValidate>
      <div>
        <label htmlFor="name">Name</label>
        <input
          id="name"
          type="text"
          value={formData.name}
          onChange={(e) => handleChange('name', e.target.value)}
          aria-invalid={!!errors.name}
          aria-describedby={errors.name ? 'name-error' : undefined}
        />
        {errors.name && <p id="name-error" role="alert">{errors.name}</p>}
      </div>

      <div>
        <label htmlFor="email">Email</label>
        <input
          id="email"
          type="email"
          value={formData.email}
          onChange={(e) => handleChange('email', e.target.value)}
          aria-invalid={!!errors.email}
        />
        {errors.email && <p role="alert">{errors.email}</p>}
      </div>

      <div>
        <label htmlFor="message">Message</label>
        <textarea
          id="message"
          value={formData.message}
          onChange={(e) => handleChange('message', e.target.value)}
          rows={4}
        />
        {errors.message && <p role="alert">{errors.message}</p>}
      </div>

      <button type="submit">Send Message</button>
    </form>
  )
}
```

| Controlled input pattern | Detail |
| ------------------------ | ------ |
| `value={state}` | React controls the input value |
| `onChange` | Updates state on every keystroke |
| `'use client'` | Required — uses useState |
| Validation before submit | Client-side for instant feedback |

Same pattern as react-project-study login and form pages — unchanged React knowledge.

# 3. Form action Attribute Basics

HTML forms can submit to a server function using the **`action`** prop — no `onSubmit` handler required.

```tsx
// app/contact/page.tsx — Server Component with form
async function submitContact(formData: FormData) {
  'use server'

  const name = formData.get('name') as string
  const email = formData.get('email') as string
  const message = formData.get('message') as string

  console.log({ name, email, message })
  // Save to database, send email, etc.
}

export default function ContactPage() {
  return (
    <main>
      <h1>Contact Us</h1>
      <form action={submitContact}>
        <div>
          <label htmlFor="name">Name</label>
          <input id="name" name="name" type="text" required />
        </div>
        <div>
          <label htmlFor="email">Email</label>
          <input id="email" name="email" type="email" required />
        </div>
        <div>
          <label htmlFor="message">Message</label>
          <textarea id="message" name="message" required />
        </div>
        <button type="submit">Send</button>
      </form>
    </main>
  )
}
```

| Form action feature | Detail |
| ------------------- | ------ |
| `action={serverFunction}` | Server Action receives `FormData` |
| `name` attribute on inputs | Keys in `FormData` |
| Works without JavaScript | Progressive enhancement |
| `'use server'` | Marks function as Server Action |

### FormData reading

```tsx
'use server'

async function createTodo(formData: FormData) {
  const title = formData.get('title')
  const priority = formData.get('priority')

  if (typeof title !== 'string' || !title.trim()) {
    return { error: 'Title is required' }
  }

  // await db.todo.create({ title, priority })
  return { success: true }
}
```

Every input needs a **`name`** attribute to appear in FormData.

# 4. Server Actions Introduction

**Server Actions** are async functions that run on the server, invoked from forms or client code.

```tsx
// actions/todos.ts
'use server'

import { revalidatePath } from 'next/cache'

export async function addTodo(formData: FormData) {
  const title = formData.get('title') as string

  if (!title?.trim()) {
    return { error: 'Title cannot be empty' }
  }

  // await saveTodoToDatabase(title)
  console.log('Saved todo:', title)

  revalidatePath('/todos')
  return { success: true }
}
```

```tsx
// app/todos/page.tsx
import { addTodo } from '@/actions/todos'

export default function TodosPage() {
  return (
    <main>
      <h1>Todos</h1>
      <form action={addTodo}>
        <input name="title" type="text" placeholder="New todo..." required />
        <button type="submit">Add</button>
      </form>
    </main>
  )
}
```

| Server Action feature | Detail |
| --------------------- | ------ |
| `'use server'` at file or function level | Marks server-only code |
| Called from forms | `action={addTodo}` |
| Called from client | `onClick={() => addTodo(formData)}` with transition |
| Returns serializable data | Can return `{ error }` or `{ success }` |
| `revalidatePath` | Refresh cached page data after mutation |

Deep Server Action patterns (validation with Zod, security) are in Topic 23.

# 5. Form Validation on Client

Client validation improves UX — show errors before a server round-trip.

```tsx
'use client'

import { useState, FormEvent } from 'react'

interface LoginFormProps {
  action: (formData: FormData) => Promise<{ error?: string }>
}

export function LoginForm({ action }: LoginFormProps) {
  const [clientError, setClientError] = useState<string | null>(null)

  async function handleSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault()
    setClientError(null)

    const formData = new FormData(event.currentTarget)
    const email = formData.get('email') as string
    const password = formData.get('password') as string

    // Client validation
    if (!email.includes('@')) {
      setClientError('Please enter a valid email')
      return
    }
    if (password.length < 6) {
      setClientError('Password must be at least 6 characters')
      return
    }

    // Pass to Server Action
    const result = await action(formData)
    if (result.error) {
      setClientError(result.error)
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      {clientError && (
        <p role="alert" className="text-red-600">{clientError}</p>
      )}
      <input name="email" type="email" placeholder="Email" required />
      <input name="password" type="password" placeholder="Password" required />
      <button type="submit">Login</button>
    </form>
  )
}
```

| Validation layer | Purpose |
| ---------------- | ------- |
| HTML attributes | `required`, `type="email"`, `minLength` — basic browser checks |
| Client JavaScript | Instant feedback, format checks |
| Server Action | **Authoritative** — never trust client alone |

Compare with react-project-study login:

```tsx
// react-project-study/src/pages/Login.tsx — client-only validation and submit
// Next.js adds server validation layer via Server Actions
```

# 6. Handling Submit States

Show loading and success feedback during form submission.

### useFormStatus (React 19 / Next.js 15)

```tsx
// components/forms/SubmitButton.tsx
'use client'

import { useFormStatus } from 'react-dom'

interface SubmitButtonProps {
  label?: string
  loadingLabel?: string
}

export function SubmitButton({
  label = 'Submit',
  loadingLabel = 'Submitting...',
}: SubmitButtonProps) {
  const { pending } = useFormStatus()

  return (
    <button type="submit" disabled={pending} aria-busy={pending}>
      {pending ? loadingLabel : label}
    </button>
  )
}
```

```tsx
// app/todos/page.tsx
import { addTodo } from '@/actions/todos'
import { SubmitButton } from '@/components/forms/SubmitButton'

export default function TodosPage() {
  return (
    <form action={addTodo}>
      <input name="title" type="text" required />
      <SubmitButton label="Add Todo" loadingLabel="Adding..." />
    </form>
  )
}
```

| Hook / pattern | Detail |
| -------------- | ------ |
| `useFormStatus()` | Must be inside `<form>` — reads pending state |
| `pending` | `true` while Server Action runs |
| `disabled={pending}` | Prevent double submit |
| Must be Client Component | `'use client'` required |

### Manual pending state with useTransition

```tsx
'use client'

import { useTransition } from 'react'
import { addTodo } from '@/actions/todos'

export function TodoForm() {
  const [isPending, startTransition] = useTransition()

  function handleSubmit(formData: FormData) {
    startTransition(async () => {
      await addTodo(formData)
    })
  }

  return (
    <form action={handleSubmit}>
      <input name="title" type="text" required />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Adding...' : 'Add Todo'}
      </button>
    </form>
  )
}
```

# 7. Revalidation After Form Submit

After a Server Action mutates data, refresh the UI with **`revalidatePath`** or **`revalidateTag`**.

```tsx
// actions/todos.ts
'use server'

import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'

export async function addTodo(formData: FormData) {
  const title = formData.get('title') as string

  if (!title?.trim()) {
    return { error: 'Title required' }
  }

  // await db.todo.create({ title })

  revalidatePath('/todos') // refresh todos page data
  return { success: true }
}

export async function deleteTodo(id: string) {
  // await db.todo.delete(id)
  revalidatePath('/todos')
}

export async function createPost(formData: FormData) {
  // await save post
  revalidatePath('/posts')
  redirect('/posts') // navigate after success
}
```

| Function | Purpose |
| -------- | ------- |
| `revalidatePath('/todos')` | Invalidate cached data for that route |
| `revalidateTag('todos')` | Invalidate all fetches tagged with 'todos' |
| `redirect('/posts')` | Navigate after successful mutation |

Without revalidation, the page shows stale data until manual refresh.

# 8. Progressive Enhancement Mindset

Server Actions work **without JavaScript** — the form submits as a regular HTML form.

```text
With JavaScript:
  User submits → Server Action via fetch → revalidate → updated UI

Without JavaScript:
  User submits → native form POST → Server Action → full page response
```

| Principle | Practice |
| --------- | -------- |
| Forms work without JS | Use `action={serverFn}` not only `onSubmit` |
| Client enhances UX | Add loading states, instant validation with JS |
| Server is authoritative | Always validate on server |
| Semantic HTML | `<form>`, `<label>`, `type`, `required` |

```tsx
// ✅ Progressive enhancement — works with and without JS
<form action={createTodo}>
  <input name="title" required />
  <button type="submit">Add</button>
</form>
```

```tsx
// ❌ JS-only — broken without JavaScript
<form onSubmit={(e) => { e.preventDefault(); /* fetch only */ }}>
```

# 9. Server Action in Separate File (Best Practice)

Keep actions in dedicated files for clarity and reuse.

```text
actions/
├── todos.ts
├── auth.ts
└── contact.ts
```

```tsx
// actions/contact.ts
'use server'

import { revalidatePath } from 'next/cache'

export interface ContactResult {
  success?: boolean
  error?: string
}

export async function submitContact(formData: FormData): Promise<ContactResult> {
  const name = formData.get('name') as string
  const email = formData.get('email') as string
  const message = formData.get('message') as string

  // Server validation — always
  if (!name?.trim()) return { error: 'Name is required' }
  if (!email?.includes('@')) return { error: 'Invalid email' }
  if (!message?.trim()) return { error: 'Message is required' }

  // Process: save, email, etc.
  console.log('Contact submission:', { name, email, message })

  revalidatePath('/contact')
  return { success: true }
}
```

```tsx
// app/contact/page.tsx
import { submitContact } from '@/actions/contact'
import { ContactFormEnhanced } from '@/components/forms/ContactFormEnhanced'

export default function ContactPage() {
  return (
    <main>
      <h1>Contact</h1>
      <ContactFormEnhanced action={submitContact} />
    </main>
  )
}
```

Same separation idea as react-project-study's `api/index.ts` — but actions run on server and can mutate data.

# 10. Returning Errors to the Client

Server Actions can return error objects displayed in the UI.

```tsx
// actions/auth.ts
'use server'

export async function login(formData: FormData) {
  const email = formData.get('email') as string
  const password = formData.get('password') as string

  if (!email || !password) {
    return { error: 'Email and password are required' }
  }

  const user = await authenticateUser(email, password)

  if (!user) {
    return { error: 'Invalid email or password' }
  }

  return { success: true }
}

async function authenticateUser(email: string, password: string) {
  // mock — replace with real auth
  if (email === 'student@training.com' && password === 'password123') {
    return { id: 1, email }
  }
  return null
}
```

```tsx
// components/forms/LoginForm.tsx
'use client'

import { useActionState } from 'react'
import { login } from '@/actions/auth'
import { SubmitButton } from './SubmitButton'

export function LoginForm() {
  const [state, formAction] = useActionState(login, { error: undefined })

  return (
    <form action={formAction}>
      {state?.error && (
        <p role="alert" className="text-red-600 bg-red-50 p-2 rounded">
          {state.error}
        </p>
      )}
      <input name="email" type="email" placeholder="Email" required />
      <input name="password" type="password" placeholder="Password" required />
      <SubmitButton label="Login" />
    </form>
  )
}
```

| Pattern | Detail |
| ------- | ------ |
| Return `{ error: string }` | Serializable — crosses server/client boundary |
| `useActionState` | React 19 hook for action state + errors |
| Never throw for expected errors | Return error object for validation failures |

# 12. Best Practices

✔ Always validate on the **server** — client validation is UX only

✔ Use **`name` attributes** on all form inputs

✔ Put Server Actions in **`actions/`** directory

✔ Call **`revalidatePath`** after mutations

✔ Use **`useFormStatus`** for submit button loading states

✔ Support **progressive enhancement** — form works without JS

✔ Return **`{ error }`** for expected failures — don't throw

✔ Associate **`<label htmlFor>`** with inputs for accessibility

✔ Prevent **double submit** with `disabled={pending}`

# 14. Summary

| Concept | Key takeaway |
| ------- | ------------ |
| Controlled inputs | `'use client'` + useState — same as React |
| Form action | `action={serverFn}` — progressive enhancement |
| Server Actions | `'use server'` functions — mutate data on server |
| Client validation | UX feedback — not security |
| Server validation | Authoritative — always required |
| Submit states | `useFormStatus`, `useTransition`, `useActionState` |
| Revalidation | `revalidatePath` after mutations |
| Progressive enhancement | Forms work without JavaScript |
| vs React | Server Actions replace manual fetch + refetch |
