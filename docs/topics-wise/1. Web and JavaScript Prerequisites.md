# Web and JavaScript Prerequisites - Teaching Notes

## Topic 1: Foundation Skills Before Next.js

---

# 1. Why This Topic Matters

Next.js is not a replacement for HTML, CSS, JavaScript, or React. It is a **full-stack React framework** that helps you build production-ready web applications with routing, server rendering, and API capabilities built in.

Before learning Next.js, students should be comfortable with:

* Reading and writing basic HTML structure
* Styling layouts with CSS
* Writing modern JavaScript (ES6+)
* Working with arrays and objects
* Fetching data with `fetch` and `async/await`
* Splitting code into modules with `import` and `export`
* **React basics:** components, JSX, props, and state (strongly recommended)

If these foundations are weak, App Router, Server Components, and Server Actions will feel confusing. This lesson fills those gaps so Next.js concepts land faster.

---

# 2. How the Web Page Works

A browser loads three main technologies together:

| Technology | Role |
| ---------- | ---- |
| HTML | Structure and content |
| CSS | Visual design and layout |
| JavaScript | Behavior and interactivity |

Next.js extends this model. It runs React on the **server** first, sends HTML to the browser, then **hydrates** interactive client components.

```text
Browser Request
       │
       ▼
  Next.js Server  ──►  Renders React (Server Components)
       │                    │
       │                    ▼
       │               Fetches data / DB
       │
       ▼
  HTML + JS sent to browser
       │
       ▼
  Client hydrates interactive ('use client') components
```

| Layer | What student already knows (React) | What Next.js adds |
| ----- | ---------------------------------- | ----------------- |
| Client | Components, hooks, browser APIs | Same — in `'use client'` files |
| Server | Not in plain React | `async` pages, Server Actions, Route Handlers |
| Routing | React Router (`BrowserRouter`) | File-based routing in `app/` folder |

# 3. HTML Basics for Next.js Developers

Next.js components return JSX that looks like HTML — same as React.

| Tag / concept | Common use in Next.js |
| ------------- | --------------------- |
| `<div>`, `<section>`, `<main>` | Layout wrappers in `layout.tsx` |
| `<h1>` – `<h6>`, `<p>` | Page content in `page.tsx` |
| `<button>`, `<input>`, `<form>` | Forms with Server Actions |
| `<ul>`, `<li>` | Lists rendered with `.map()` |
| `<img>`, `<a>` | Prefer `next/image` and `next/link` later |
| `alt`, `type`, `placeholder` | Accessibility and form behavior |

### Semantic HTML

Semantic tags describe meaning, not just appearance.

```html
<header>
  <h1>Training Portal</h1>
</header>

<main>
  <article>
    <h2>Next.js Basics</h2>
    <p>Learn App Router, layouts, and Server Components.</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 Training Team</p>
</footer>
```

### Why semantics matter

* Screen readers understand page structure better
* SEO improves — Next.js `generateMetadata` works best with meaningful page structure
* Server-rendered HTML is indexed by search engines

# 4. CSS Basics for Next.js Developers

Next.js does not remove the need for CSS. Most apps use Tailwind CSS, CSS Modules, or global CSS.

| CSS topic | Why it matters in Next.js |
| --------- | -------------------------- |
| Box model | Padding and margin inside components |
| Flexbox | Nav bars, card rows, centered layouts |
| Grid | Dashboards and gallery layouts |
| Classes | Applied via `className` in JSX |
| Responsive design | Mobile-first UI — critical for Core Web Vitals |

### Flexbox example

```css
.product-list {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.product-card {
  flex: 1 1 240px;
  padding: 16px;
  border: 1px solid #ddd;
  border-radius: 8px;
}
```

### Global CSS in Next.js

In a Next.js project, global styles live in `app/globals.css` and are imported once in the root layout:

```tsx
// app/layout.tsx
import './globals.css'
```

Component-scoped styles use **CSS Modules**: `Button.module.css` imported as `styles.button`.

# 5. Modern JavaScript: Variables and Functions

Next.js code is written in modern JavaScript (ES6+) or TypeScript.

### `let` and `const`

```javascript
const appName = "Next.js Training";
let activeTab = "home";

activeTab = "products";
// appName = "Other"; // Error: const cannot be reassigned
```

| Keyword | Use |
| ------- | --- |
| `const` | Values that should not be reassigned (default choice) |
| `let` | Values that change over time |

### Arrow functions

Components are usually arrow functions or `function` declarations.

```javascript
const double = (value) => value * 2;

function greetUser(name) {
  return `Hello, ${name}`;
}

console.log(double(5));        // 10
console.log(greetUser("Anu")); // Hello, Anu
```

### Template literals

```javascript
const user = { name: "Ravi", role: "Admin" };
const message = `${user.name} is logged in as ${user.role}`;
```

# 6. Objects, Destructuring, and Spread

React props, Next.js `params`, and `searchParams` all use objects.

### Object basics

```javascript
const product = {
  id: 1,
  name: "Keyboard",
  price: 1499,
  inStock: true,
};

console.log(product.name);
console.log(product["price"]);
```

### Destructuring (used everywhere in Next.js)

```javascript
const { name, price } = product;

// Similar to how page components receive params:
function ProductPage({ params }) {
  const { id } = params;
  return `Product ID: ${id}`;
}
```

### Spread operator

```javascript
const updatedProduct = {
  ...product,
  price: 1299,
};

const tags = ["electronics", "accessories"];
const allTags = [...tags, "keyboard"];
```

| Operator | Purpose |
| -------- | ------- |
| `{ ...obj }` | Copy or update object fields |
| `[...arr]` | Copy or merge arrays |

# 7. Array Methods Used in Next.js

Lists in Next.js pages are built from arrays using `.map()`, `.filter()`, and related methods — same as React.

### `.map()` — transform each item

```javascript
const users = [
  { id: 1, name: "Anu", isActive: true },
  { id: 2, name: "Ravi", isActive: false },
  { id: 3, name: "Meera", isActive: true },
];

const activeUserNames = users
  .filter((user) => user.isActive)
  .map((user) => user.name);

console.log(activeUserNames); // ["Anu", "Meera"]
```

| Method | Returns | Typical Next.js use |
| ------ | ------- | ------------------- |
| `.map()` | New array (same length) | Render a list in `page.tsx` |
| `.filter()` | New array (subset) | Filter products by category |
| `.find()` | One item or `undefined` | Get record by id from params |
| `.reduce()` | Single value | Totals, counts, grouping |
| `.some()` / `.every()` | Boolean | Validation checks |

### Rendering in a Server Component (preview)

```tsx
// app/users/page.tsx — async server component (Topic 10)
export default async function UsersPage() {
  const users = await loadUsers();

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

# 8. Conditional Logic and Short-Circuit Patterns

JSX expressions cannot contain full `if` statements, but JavaScript conditionals are used everywhere.

### `if` / `else`

```javascript
function getStatusLabel(isLoggedIn) {
  if (isLoggedIn) {
    return "Welcome back";
  }
  return "Please sign in";
}
```

### Ternary operator

```javascript
const label = isLoggedIn ? "Logout" : "Login";
```

### Logical AND (`&&`)

```javascript
const errorMessage = "";
const showError = errorMessage && `Error: ${errorMessage}`;
```

These patterns appear in Next.js for loaders, errors, empty states, and conditional UI in both server and client components.

# 9. Promises and Async/Await

Next.js Server Components and Route Handlers are **async** by design. Students must understand asynchronous JavaScript.

### Promise with `.then()`

```javascript
fetch("https://jsonplaceholder.typicode.com/users")
  .then((response) => {
    if (!response.ok) {
      throw new Error("Network response was not ok");
    }
    return response.json();
  })
  .then((users) => {
    console.log(users);
  })
  .catch((error) => {
    console.error("Failed to load users", error);
  });
```

### `async` / `await` (preferred — used in Next.js pages)

```javascript
async function loadUsers() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users");

    if (!response.ok) {
      throw new Error("Could not fetch users");
    }

    return await response.json();
  } catch (error) {
    console.error("API error:", error.message);
    return [];
  }
}
```

### How this maps to Next.js

```tsx
// Server Component — fetch runs on the server, not in the browser
export default async function UsersPage() {
  const users = await loadUsers();
  return <div>{users.length} users loaded</div>;
}
```

| Concept | React (Vite) | Next.js |
| ------- | ------------ | ------- |
| Data fetch timing | `useEffect` on mount | `async` page / server function |
| Where code runs | Browser only | Server first, then client |
| Loading state | `useState` + manual | `loading.tsx` + Suspense |

# 10. JavaScript Modules

Next.js projects split code into multiple files using ES modules — same as React.

### Named exports

```javascript
// lib/formatPrice.ts
export function formatPrice(amount: number) {
  return `Rs. ${amount.toLocaleString("en-IN")}`;
}

export const TAX_RATE = 0.18;
```

### Default export

```javascript
// components/ProductCard.tsx
export default function ProductCard({ name, price }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>{price}</p>
    </div>
  );
}
```

### Imports

```typescript
import ProductCard from "@/components/ProductCard";
import { formatPrice, TAX_RATE } from "@/lib/formatPrice";
```

| Pattern | Syntax | When to use |
| ------- | ------ | ----------- |
| Named export | `export function X()` | Utilities, multiple exports per file |
| Default export | `export default X` | One main thing per file (e.g. page, component) |
| Named import | `import { X } from "./file"` | Must match export name |
| Default import | `import X from "./file"` | Any local name is allowed |

Next.js also supports the `@/` path alias (points to project root or `src/`).

# 11. React Basics Recap (Required for Next.js)

Students coming from the React syllabus should know these before Topic 2:

| React concept | Next.js equivalent |
| ------------- | ------------------ |
| `function App()` | `app/page.tsx` default export |
| `<BrowserRouter>` | File-based routing — no manual router setup |
| `useState` | Client Components only (`'use client'`) |
| `useEffect` + fetch | Prefer `async` Server Components |
| Props | Same — passed between components |
| `children` | Used heavily in `layout.tsx` |

### Plain React entry (react-project-study)

```tsx
// src/main.tsx — React with Vite needs manual router setup
createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>,
)
```

### Next.js entry (no manual `createRoot` — framework handles it)

```tsx
// app/layout.tsx — root layout wraps all pages
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

# 12. Best Practices Before Next.js

✔ Use `const` by default; use `let` only when reassignment is needed

✔ Prefer arrow functions for short callbacks (e.g. inside `.map()`)

✔ Use destructuring for props, `params`, and `searchParams`

✔ Keep data transformations pure — `.map()` and `.filter()` should not mutate the original array

✔ Always handle API errors with `try/catch` or `.catch()`

✔ Use meaningful variable names (`isLoading`, `hasError`, `activeUsers`)

✔ Prefer semantic HTML tags over endless `<div>` soup

✔ Complete React topics 1–11 before deep-diving into Server Components

✔ Understand the difference between **browser JavaScript** and **server JavaScript**
