# Dynamic Routes and Params - Teaching Notes

## Topic 16: URL Segments, Params, and Static Generation

---

# 1. Why This Topic Matters

Real apps rarely have only static URLs like `/about` or `/contact`. You need routes like `/products/42`, `/blog/my-first-post`, or `/docs/getting-started/installation`.

Dynamic routes let folder names in brackets capture URL segments. The captured values arrive as **`params`** in your page and layout components. Understanding dynamic routing is essential for product pages, user profiles, blog posts, and admin panels.

---

# 2. Dynamic Segments

Use square brackets in folder names to create dynamic segments.

```text
app/
  products/
    page.tsx              → /products
    [id]/
      page.tsx            → /products/42, /products/abc
  blog/
    [slug]/
      page.tsx            → /blog/hello-world
  shop/
    [category]/
      [productId]/
        page.tsx          → /shop/electronics/phone-12
```

```tsx
// app/products/[id]/page.tsx
interface ProductPageProps {
  params: Promise<{ id: string }>
}

export default async function ProductPage({ params }: ProductPageProps) {
  const { id } = await params

  return (
    <main>
      <h1>Product Details</h1>
      <p>Showing product ID: {id}</p>
    </main>
  )
}
```

| URL | `params.id` |
| --- | ----------- |
| `/products/42` | `"42"` |
| `/products/keyboard-pro` | `"keyboard-pro"` |

**Note (Next.js 15+):** `params` is a **Promise** — always `await params` before destructuring.

# 3. Catch-All Routes

Capture **multiple** path segments with `[...slug]`.

```text
app/
  docs/
    [...slug]/
      page.tsx    → /docs/a, /docs/a/b, /docs/a/b/c
```

```tsx
// app/docs/[...slug]/page.tsx
interface DocsPageProps {
  params: Promise<{ slug: string[] }>
}

export default async function DocsPage({ params }: DocsPageProps) {
  const { slug } = await params

  return (
    <main>
      <h1>Documentation</h1>
      <p>Path segments: {slug.join(' / ')}</p>
      <nav aria-label="Breadcrumb">
        {slug.map((segment, index) => (
          <span key={index}>
            {index > 0 && ' → '}
            {segment}
          </span>
        ))}
      </nav>
    </main>
  )
}
```

| URL | `params.slug` |
| --- | ------------- |
| `/docs/intro` | `["intro"]` |
| `/docs/guides/setup` | `["guides", "setup"]` |
| `/docs/a/b/c/d` | `["a", "b", "c", "d"]` |

# 4. Optional Catch-All Routes

Optional catch-all `[[...slug]]` also matches the **base path** without any segments.

```text
app/
  shop/
    [[...slug]]/
      page.tsx    → /shop AND /shop/electronics AND /shop/electronics/phones
```

```tsx
// app/shop/[[...slug]]/page.tsx
interface ShopPageProps {
  params: Promise<{ slug?: string[] }>
}

export default async function ShopPage({ params }: ShopPageProps) {
  const { slug } = await params

  if (!slug || slug.length === 0) {
    return <h1>Shop — All Categories</h1>
  }

  if (slug.length === 1) {
    return <h1>Category: {slug[0]}</h1>
  }

  return (
    <h1>
      Product: {slug[1]} in {slug[0]}
    </h1>
  )
}
```

| Pattern | Matches base path? | Example |
| ------- | ------------------ | ------- |
| `[slug]` | ❌ No | `/blog/post` only |
| `[...slug]` | ❌ No | `/docs/a/b` — not `/docs` |
| `[[...slug]]` | ✅ Yes | `/shop` and `/shop/electronics` |

# 5. Using Params to Fetch Data

The most common pattern: use the dynamic segment to load a specific record.

```tsx
// lib/posts.ts
export interface Post {
  slug: string
  title: string
  content: string
  publishedAt: string
}

export async function getPostBySlug(slug: string): Promise<Post | null> {
  const res = await fetch(`https://api.example.com/posts/${slug}`, {
    next: { tags: [`post-${slug}`] },
  })

  if (res.status === 404) return null
  if (!res.ok) throw new Error(`Failed to fetch post: ${slug}`)

  return res.json()
}
```

```tsx
// app/blog/[slug]/page.tsx
import { notFound } from 'next/navigation'
import { getPostBySlug } from '@/lib/posts'

interface BlogPostPageProps {
  params: Promise<{ slug: string }>
}

export default async function BlogPostPage({ params }: BlogPostPageProps) {
  const { slug } = await params
  const post = await getPostBySlug(slug)

  if (!post) {
    notFound() // Renders not-found.tsx
  }

  return (
    <article>
      <h1>{post.title}</h1>
      <time dateTime={post.publishedAt}>
        {new Date(post.publishedAt).toLocaleDateString('en-IN')}
      </time>
      <div>{post.content}</div>
    </article>
  )
}
```

| Step | Action |
| ---- | ------ |
| 1 | Await and destructure `params` |
| 2 | Fetch data using param value |
| 3 | Call `notFound()` if record missing |
| 4 | Render page with fetched data |

# 6. generateStaticParams

Pre-render dynamic routes at **build time** for static generation (SSG).

```tsx
// app/products/[id]/page.tsx
import { getAllProductIds, getProductById } from '@/lib/products'

export async function generateStaticParams() {
  const products = await getAllProductIds()

  return products.map((product) => ({
    id: product.id,
  }))
}

interface ProductPageProps {
  params: Promise<{ id: string }>
}

export default async function ProductPage({ params }: ProductPageProps) {
  const { id } = await params
  const product = await getProductById(id)

  if (!product) notFound()

  return (
    <main>
      <h1>{product.name}</h1>
      <p>Rs. {product.price}</p>
    </main>
  )
}
```

```tsx
// lib/products.ts
export async function getAllProductIds() {
  const res = await fetch('https://api.example.com/products')
  const products = await res.json()
  return products.map((p: { id: string }) => ({ id: p.id }))
}
```

| Without `generateStaticParams` | With `generateStaticParams` |
| ------------------------------ | --------------------------- |
| Page rendered on first request | Pages pre-built at `npm run build` |
| Slower first visit | Faster — served as static HTML |
| Good for unknown/infinite IDs | Good for known, finite set of IDs |

# 7. Dynamic vs Static Route Decisions

| Scenario | Recommendation |
| -------- | -------------- |
| Blog posts (known at build) | `generateStaticParams` + ISR |
| User profiles (millions of users) | Dynamic — render on request |
| Product catalog (updates often) | `generateStaticParams` + `revalidate` |
| Admin-only pages | Dynamic with auth check |
| Docs with hundreds of pages | Static generation with catch-all |

```tsx
// Force dynamic rendering for user-specific pages
export const dynamic = 'force-dynamic'

// Or set revalidation interval
export const revalidate = 3600 // revalidate every hour
```

| Export | Effect |
| ------ | ------ |
| `generateStaticParams()` | Pre-render listed params at build |
| `export const revalidate = 60` | ISR — regenerate every 60 seconds |
| `export const dynamic = 'force-dynamic'` | Always server-render on each request |
| `fetch(..., { cache: 'no-store' })` | Opt out of static caching for that data |

# 8. Params in Layouts and Route Handlers

Dynamic params are available in layouts and route handlers too.

```tsx
// app/shop/[category]/layout.tsx
interface CategoryLayoutProps {
  children: React.ReactNode
  params: Promise<{ category: string }>
}

export default async function CategoryLayout({
  children,
  params,
}: CategoryLayoutProps) {
  const { category } = await params

  return (
    <div>
      <header>
        <h1>Category: {category.replace('-', ' ')}</h1>
      </header>
      {children}
    </div>
  )
}
```

```tsx
// app/api/users/[id]/route.ts
import { NextRequest, NextResponse } from 'next/server'

interface RouteContext {
  params: Promise<{ id: string }>
}

export async function GET(_request: NextRequest, context: RouteContext) {
  const { id } = await context.params
  const user = await db.user.findUnique({ where: { id } })

  if (!user) {
    return NextResponse.json({ error: 'User not found' }, { status: 404 })
  }

  return NextResponse.json(user)
}
```

# 9. Typing Params with TypeScript

Use interfaces for type-safe params across pages and layouts.

```tsx
// types/routes.ts
export interface ProductParams {
  id: string
}

export interface DocsParams {
  slug: string[]
}

export interface ShopParams {
  slug?: string[]
}
```

```tsx
// app/products/[id]/page.tsx
import type { ProductParams } from '@/types/routes'

interface ProductPageProps {
  params: Promise<ProductParams>
}

export default async function ProductPage({ params }: ProductPageProps) {
  const { id } = await params
  // id is typed as string
}
```

| Route folder | Params type |
| ------------ | ----------- |
| `[id]` | `{ id: string }` |
| `[...slug]` | `{ slug: string[] }` |
| `[[...slug]]` | `{ slug?: string[] }` |
| `[category]/[productId]` | `{ category: string; productId: string }` |

# 10. Not Found Handling

Always handle invalid params gracefully.

```tsx
import { notFound } from 'next/navigation'

export default async function ProductPage({ params }: ProductPageProps) {
  const { id } = await params

  // Validate param format
  if (!/^[a-zA-Z0-9-]+$/.test(id)) {
    notFound()
  }

  const product = await getProductById(id)

  if (!product) {
    notFound()
  }

  return <div>{product.name}</div>
}
```

```tsx
// app/products/[id]/not-found.tsx
export default function ProductNotFound() {
  return (
    <main>
      <h1>Product Not Found</h1>
      <p>The product you are looking for does not exist or was removed.</p>
      <a href="/products">Browse all products</a>
    </main>
  )
}
```

| Function / file | Scope |
| --------------- | ----- |
| `notFound()` | Triggers nearest `not-found.tsx` |
| `app/not-found.tsx` | Global 404 UI |
| `app/products/[id]/not-found.tsx` | Product-specific 404 |

# 11. Best Practices

✔ Always `await params` in Next.js 15+ before using values

✔ Validate param format before querying the database

✔ Call `notFound()` for missing resources — don't show empty pages

✔ Use `generateStaticParams` for known, finite route sets

✔ Combine static generation with ISR for content that updates periodically

✔ Type params with interfaces for safer refactoring

✔ Keep fetch logic in `lib/` — pages should focus on rendering
