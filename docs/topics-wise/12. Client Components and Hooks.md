# Client Components and Hooks - Teaching Notes

## Topic 12: useState, useEffect, and Interactive UI in Next.js

---

# 1. Why This Topic Matters

Server Components handle data and static UI. **Client Components** handle everything interactive — clicks, forms, animations, browser APIs, and React hooks.

Students who mark entire pages `'use client'` defeat the purpose of Next.js. This topic teaches:

* When `'use client'` is truly required
* How to use hooks correctly inside Client Components
* Rules of Hooks still apply
* How to minimize client bundle size while building rich UI

Hooks are the same as react-project-study — the new skill is **knowing where** to use them.

---

# 2. The 'use client' Directive

Place **`'use client'`** as the first line of a file to mark it and its imports as client-side.

```tsx
// components/Counter.tsx
'use client'

import { useState } from 'react'

export function Counter() {
  const [count, setCount] = useState(0)

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  )
}
```

| Rule | Detail |
| ---- | ------ |
| File-level directive | Applies to entire file |
| Must be first line | Before any imports |
| Creates client boundary | Everything imported becomes client bundle |
| Does not propagate up | Parent Server Component stays server |

```tsx
// app/page.tsx — Server Component (no directive)
import { Counter } from '@/components/Counter'

export default function HomePage() {
  return (
    <main>
      <h1>Home</h1>
      <Counter /> {/* Client island in server page */}
    </main>
  )
}
```

# 3. useState in Next.js

**`useState`** works identically to react-project-study — only inside Client Components.

```tsx
'use client'

import { useState } from 'react'

export function TogglePanel() {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <div>
      <button
        onClick={() => setIsOpen(!isOpen)}
        aria-expanded={isOpen}
      >
        {isOpen ? 'Collapse' : 'Expand'}
      </button>
      {isOpen && (
        <div className="mt-2 p-4 border rounded">
          <p>Panel content visible when expanded.</p>
        </div>
      )}
    </div>
  )
}
```

### Initial state from server props

```tsx
'use client'

import { useState } from 'react'

interface EditableTitleProps {
  initialTitle: string
}

export function EditableTitle({ initialTitle }: EditableTitleProps) {
  const [title, setTitle] = useState(initialTitle)
  const [isEditing, setIsEditing] = useState(false)

  if (!isEditing) {
    return (
      <div className="flex items-center gap-2">
        <h2>{title}</h2>
        <button onClick={() => setIsEditing(true)}>Edit</button>
      </div>
    )
  }

  return (
    <input
      value={title}
      onChange={(e) => setTitle(e.target.value)}
      onBlur={() => setIsEditing(false)}
      autoFocus
    />
  )
}
```

Server Component passes serializable `initialTitle` — client manages edits locally.

| useState in Next.js | Detail |
| ------------------- | ------ |
| Client only | Server Components cannot use useState |
| Props as initial values | Pass data from server fetch as props |
| State is local | Lost on full page refresh unless persisted |

# 4. useEffect in Next.js

**`useEffect`** runs after render in the browser — use for side effects that cannot run on the server.

```tsx
'use client'

import { useState, useEffect } from 'react'

export function Clock() {
  const [time, setTime] = useState<string>('')

  useEffect(() => {
    function updateTime() {
      setTime(new Date().toLocaleTimeString('en-IN'))
    }

    updateTime()
    const interval = setInterval(updateTime, 1000)

    return () => clearInterval(interval) // cleanup
  }, [])

  return <p>Current time: {time}</p>
}
```

### Common useEffect uses in Client Components

| Use case | Example |
| -------- | ------- |
| Browser APIs | `localStorage`, `window` size, geolocation |
| Subscriptions | WebSocket, event listeners |
| Client-only fetch | User-triggered refresh, infinite scroll |
| Third-party libraries | Charts, maps requiring DOM |
| Sync with localStorage | Theme preference, cart persistence |

### When NOT to use useEffect in Next.js

```tsx
// ❌ Avoid — fetch initial page data in useEffect
'use client'
export default function ProductsPage() {
  const [products, setProducts] = useState([])
  useEffect(() => {
    fetch('/api/products').then(r => r.json()).then(setProducts)
  }, [])
}

// ✅ Prefer — Server Component fetch
export default async function ProductsPage() {
  const products = await getProducts()
  return <ProductList products={products} />
}
```

Compare with react-project-study where useEffect fetch is the standard pattern:

```tsx
// react-project-study/src/pages/DataFetch.tsx
useEffect(() => {
  fetchData()
}, [])
```

In Next.js, reserve useEffect for **client-only side effects** — not initial page data.

# 5. useRef

**`useRef`** holds mutable values that do not trigger re-renders — DOM references and persistent values.

```tsx
'use client'

import { useRef, useEffect } from 'react'

export function SearchInput() {
  const inputRef = useRef<HTMLInputElement>(null)

  useEffect(() => {
    inputRef.current?.focus()
  }, [])

  return (
    <input
      ref={inputRef}
      type="search"
      placeholder="Search..."
      aria-label="Search"
    />
  )
}
```

```tsx
'use client'

import { useRef } from 'react'

export function Stopwatch() {
  const intervalRef = useRef<NodeJS.Timeout | null>(null)
  const secondsRef = useRef(0)

  function start() {
    if (intervalRef.current) return
    intervalRef.current = setInterval(() => {
      secondsRef.current += 1
      console.log(secondsRef.current)
    }, 1000)
  }

  function stop() {
    if (intervalRef.current) {
      clearInterval(intervalRef.current)
      intervalRef.current = null
    }
  }

  return (
    <div>
      <button onClick={start}>Start</button>
      <button onClick={stop}>Stop</button>
    </div>
  )
}
```

| useRef use | Detail |
| ---------- | ------ |
| DOM access | `ref={inputRef}` — focus, measure, scroll |
| Mutable value | Store interval ID without re-render |
| Client only | Requires browser DOM |

Same as react-project-study `ReactRef.tsx` — no changes in Next.js.

# 6. useContext

**`useContext`** shares state across Client Components without prop drilling.

```tsx
// components/providers/ThemeProvider.tsx
'use client'

import { createContext, useContext, useState, ReactNode } from 'react'

type Theme = 'light' | 'dark'

interface ThemeContextValue {
  theme: Theme
  toggleTheme: () => void
}

const ThemeContext = createContext<ThemeContextValue | null>(null)

export function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setTheme] = useState<Theme>('light')

  function toggleTheme() {
    setTheme((prev) => (prev === 'light' ? 'dark' : 'light'))
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      <div className={theme}>{children}</div>
    </ThemeContext.Provider>
  )
}

export function useTheme() {
  const context = useContext(ThemeContext)
  if (!context) {
    throw new Error('useTheme must be used within ThemeProvider')
  }
  return context
}
```

```tsx
// app/layout.tsx — Server Component wraps client provider
import { ThemeProvider } from '@/components/providers/ThemeProvider'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  )
}
```

```tsx
// components/ThemeToggle.tsx
'use client'

import { useTheme } from '@/components/providers/ThemeProvider'

export function ThemeToggle() {
  const { theme, toggleTheme } = useTheme()

  return (
    <button onClick={toggleTheme} aria-label={`Switch to ${theme === 'light' ? 'dark' : 'light'} mode`}>
      {theme === 'light' ? '🌙' : '☀️'}
    </button>
  )
}
```

| Context in Next.js | Detail |
| ------------------ | ------ |
| Provider must be Client Component | `'use client'` on provider file |
| Can wrap in root layout | Server layout imports client provider |
| Children can be Server Components | Passed as children — composition pattern |

Same pattern as react-project-study `ReactContext.tsx`.

# 7. useMemo and useCallback

Optimize re-renders by memoizing expensive computations and stable function references.

```tsx
'use client'

import { useState, useMemo, useCallback } from 'react'

interface Product {
  id: number
  name: string
  price: number
  category: string
}

interface ProductFilterProps {
  products: Product[]
}

export function ProductFilter({ products }: ProductFilterProps) {
  const [search, setSearch] = useState('')
  const [category, setCategory] = useState('all')

  // Memoize filtered list — recalculate only when deps change
  const filteredProducts = useMemo(() => {
    return products.filter((product) => {
      const matchesSearch = product.name.toLowerCase().includes(search.toLowerCase())
      const matchesCategory = category === 'all' || product.category === category
      return matchesSearch && matchesCategory
    })
  }, [products, search, category])

  // Stable callback reference for child components
  const handleSelect = useCallback((id: number) => {
    console.log('Selected product:', id)
  }, [])

  return (
    <div>
      <input
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search..."
      />
      <select value={category} onChange={(e) => setCategory(e.target.value)}>
        <option value="all">All</option>
        <option value="electronics">Electronics</option>
        <option value="books">Books</option>
      </select>
      <ul>
        {filteredProducts.map((product) => (
          <li key={product.id} onClick={() => handleSelect(product.id)}>
            {product.name} — Rs. {product.price}
          </li>
        ))}
      </ul>
    </div>
  )
}
```

| Hook | Purpose | When to use |
| ---- | ------- | ----------- |
| `useMemo` | Cache computed value | Expensive filtering, sorting, transformations |
| `useCallback` | Cache function reference | Pass stable callbacks to memoized children |

Same as react-project-study `UesMemoTest.tsx` and FilterableProductTable patterns.

**Do not overuse** — memoization has its own cost. Apply when profiling shows re-render problems.

# 8. Rules of Hooks in Client Components

All React Rules of Hooks apply unchanged in Next.js Client Components.

| Rule | Detail |
| ---- | ------ |
| Top level only | Never call hooks inside conditions, loops, or nested functions |
| Client Components only | Hooks never in Server Components |
| Consistent order | Same hooks in same order every render |
| Custom hooks start with `use` | `useCart`, `useDebounce` |

```tsx
// ❌ Wrong — conditional hook
'use client'
function BadComponent({ isLoggedIn }: { isLoggedIn: boolean }) {
  if (isLoggedIn) {
    const [user, setUser] = useState(null) // Rules of Hooks violation
  }
}

// ✅ Correct — hook at top level
'use client'
function GoodComponent({ isLoggedIn }: { isLoggedIn: boolean }) {
  const [user, setUser] = useState(null)

  useEffect(() => {
    if (isLoggedIn) {
      fetchUser().then(setUser)
    }
  }, [isLoggedIn])
}
```

Custom hooks extract reusable client logic:

```tsx
// hooks/useDebounce.ts
'use client'

import { useState, useEffect } from 'react'

export function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay)
    return () => clearTimeout(timer)
  }, [value, delay])

  return debouncedValue
}
```

# 9. When Client Components Are Necessary

| Need | Client Component? |
| ---- | ----------------- |
| `useState`, `useEffect`, any hook | ✅ Yes |
| `onClick`, `onChange`, event handlers | ✅ Yes |
| `window`, `document`, `localStorage` | ✅ Yes |
| Browser-only libraries (charts, maps) | ✅ Yes |
| Static text and images from server data | ❌ Server Component |
| async data fetch for page | ❌ Server Component |
| Server Actions as form action | ❌ Server (action) + optional client for UX |

### Minimize client boundary

```text
❌ Large client page
app/dashboard/page.tsx ('use client' — entire page)

✅ Small client islands
app/dashboard/page.tsx (Server — fetches data)
  └── DashboardChart.tsx ('use client' — chart only)
  └── StatsSummary.tsx (Server — static stats)
  └── DateRangePicker.tsx ('use client' — interactive picker)
```

# 10. Minimizing Client Bundle Size

Each `'use client'` file adds JavaScript sent to the browser.

| Strategy | Detail |
| -------- | ------ |
| Leaf-level client components | Only interactive parts are client |
| Server fetch + client display | Data from server props — no client fetch library |
| Dynamic import | Lazy load heavy client components |
| Avoid barrel imports | Import specific components, not entire libraries |
| Split providers | One provider per concern — not one mega provider |

### Dynamic import for heavy client components

```tsx
// app/analytics/page.tsx — Server Component
import dynamic from 'next/dynamic'

const AnalyticsChart = dynamic(
  () => import('@/components/analytics/AnalyticsChart'),
  {
    loading: () => <p>Loading chart...</p>,
    ssr: false, // skip server render for browser-only chart
  },
)

export default async function AnalyticsPage() {
  const data = await getAnalyticsData()
  return (
    <main>
      <h1>Analytics</h1>
      <AnalyticsChart data={data} />
    </main>
  )
}
```

| dynamic import option | Purpose |
| --------------------- | ------- |
| `loading` | Fallback while chunk loads |
| `ssr: false` | Client-only render (charts, maps) |

# 11. Interactive UI Patterns

### Pattern A — Tabs

```tsx
'use client'

import { useState } from 'react'
import clsx from 'clsx'

interface Tab {
  id: string
  label: string
}

interface TabsProps {
  tabs: Tab[]
  children: React.ReactNode[]
}

export function Tabs({ tabs, children }: TabsProps) {
  const [activeTab, setActiveTab] = useState(tabs[0]?.id ?? '')

  const activeIndex = tabs.findIndex((t) => t.id === activeTab)

  return (
    <div>
      <div role="tablist" className="flex gap-2 border-b">
        {tabs.map((tab) => (
          <button
            key={tab.id}
            role="tab"
            aria-selected={activeTab === tab.id}
            onClick={() => setActiveTab(tab.id)}
            className={clsx(
              'px-4 py-2',
              activeTab === tab.id && 'border-b-2 border-brand-500 font-semibold',
            )}
          >
            {tab.label}
          </button>
        ))}
      </div>
      <div role="tabpanel" className="pt-4">
        {children[activeIndex]}
      </div>
    </div>
  )
}
```

### Pattern B — Modal

```tsx
'use client'

import { useState, useEffect } from 'react'

interface ModalProps {
  trigger: React.ReactNode
  children: React.ReactNode
  title: string
}

export function Modal({ trigger, children, title }: ModalProps) {
  const [isOpen, setIsOpen] = useState(false)

  useEffect(() => {
    function handleEscape(event: KeyboardEvent) {
      if (event.key === 'Escape') setIsOpen(false)
    }
    if (isOpen) {
      document.addEventListener('keydown', handleEscape)
      return () => document.removeEventListener('keydown', handleEscape)
    }
  }, [isOpen])

  return (
    <>
      <div onClick={() => setIsOpen(true)}>{trigger}</div>
      {isOpen && (
        <div
          className="fixed inset-0 bg-black/50 flex items-center justify-center"
          role="dialog"
          aria-modal="true"
          aria-labelledby="modal-title"
        >
          <div className="bg-white rounded-lg p-6 max-w-md w-full">
            <h2 id="modal-title" className="text-lg font-bold mb-4">{title}</h2>
            {children}
            <button onClick={() => setIsOpen(false)} className="mt-4">
              Close
            </button>
          </div>
        </div>
      )}
    </>
  )
}
```

### Pattern C — Client fetch on user action

```tsx
'use client'

import { useState } from 'react'

export function RefreshableData() {
  const [data, setData] = useState<string[]>([])
  const [isLoading, setIsLoading] = useState(false)

  async function handleRefresh() {
    setIsLoading(true)
    try {
      const res = await fetch('/api/items')
      const items = await res.json()
      setData(items)
    } finally {
      setIsLoading(false)
    }
  }

  return (
    <div>
      <button onClick={handleRefresh} disabled={isLoading}>
        {isLoading ? 'Loading...' : 'Refresh'}
      </button>
      <ul>
        {data.map((item, i) => (
          <li key={i}>{item}</li>
        ))}
      </ul>
    </div>
  )
}
```

| Hook / pattern | react-project-study | Next.js Client Component |

| useState | Everywhere | Client Components only |

| useContext | ReactContext.tsx | Provider in layout — same API |

| Component default | All client | Opt-in with `'use client'` |

```tsx

// Next.js: same hooks, but isolated in 'use client' files
```

The hooks API is **identical**. The architectural change is **where** they run.

# 13. Best Practices

✔ Add **`'use client'`** only to files that need hooks or browser APIs

✔ Keep client components **small** — leaf nodes in the tree

✔ Fetch **initial data on server** — use client fetch for user-triggered updates

✔ Place **providers** in client wrapper imported by root layout

✔ Follow **Rules of Hooks** — top level, consistent order

✔ Use **`dynamic()`** for heavy client-only libraries

✔ Extract reusable logic into **custom hooks** in `hooks/` folder

✔ Clean up **useEffect** subscriptions and timers

✔ Do not convert Server Components to client just to use one hook — extract instead
