# Middleware - Teaching Notes

## Topic 18: Request Interception, Auth, and Edge Runtime

---

# 1. Why This Topic Matters

Middleware runs **before** a request reaches your page or API route. It sits between the user and your app — perfect for authentication checks, redirects, locale detection, A/B testing, and rewriting URLs.

Without middleware, you'd duplicate auth logic in every protected page. With middleware, you centralize checks once and protect entire route groups efficiently.

---

# 2. What Is Next.js Middleware?

Middleware is a function in `middleware.ts` at the **project root** (or inside `src/` if using src directory).

```text
my-app/
  middleware.ts     ← project root (or src/middleware.ts)
  app/
    dashboard/
      page.tsx
    login/
      page.tsx
  ...
```

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  console.log('Middleware hit:', request.nextUrl.pathname)
  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/admin/:path*'],
}
```

```text
Incoming Request
       │
       ▼
  middleware.ts     ← runs first (Edge Runtime)
       │
       ├── redirect / rewrite / modify headers
       │
       ▼
  page.tsx / route.ts
```

| Feature | Detail |
| ------- | ------ |
| Runs on | Edge Runtime (lightweight, globally distributed) |
| Timing | Before route handler or page renders |
| Can do | Redirect, rewrite, set headers, read cookies |
| Cannot do | Heavy DB queries, Node.js-only APIs, render React |

# 3. The Matcher Config

Control which paths trigger middleware with `config.matcher`.

```tsx
// middleware.ts
export const config = {
  matcher: [
    // Protect dashboard and all sub-routes
    '/dashboard/:path*',

    // Protect admin section
    '/admin/:path*',

    // API routes that need auth
    '/api/protected/:path*',
  ],
}
```

```tsx
// Exclude static files and Next.js internals
export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico|public).*)',
  ],
}
```

| Pattern | Matches |
| ------- | ------- |
| `/dashboard` | Exact path only |
| `/dashboard/:path*` | Dashboard and all nested paths |
| `/api/:path*` | All API routes |
| Negative lookahead `(?!...)` | Exclude patterns (static assets) |

# 4. Redirects in Middleware

Redirect unauthenticated users before the page loads.

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

const PUBLIC_PATHS = ['/login', '/register', '/forgot-password']

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // Allow public paths through
  if (PUBLIC_PATHS.some((path) => pathname.startsWith(path))) {
    return NextResponse.next()
  }

  const sessionToken = request.cookies.get('session')?.value

  if (!sessionToken) {
    const loginUrl = new URL('/login', request.url)
    loginUrl.searchParams.set('callbackUrl', pathname)
    return NextResponse.redirect(loginUrl)
  }

  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/profile/:path*', '/settings/:path*'],
}
```

| Response | Method |
| -------- | ------ |
| Continue to page | `NextResponse.next()` |
| Temporary redirect | `NextResponse.redirect(url)` |
| Permanent redirect | `NextResponse.redirect(url, 308)` |
| Rewrite (URL bar unchanged) | `NextResponse.rewrite(url)` |

# 5. Reading Cookies and Headers

Middleware can inspect request cookies and headers for auth and locale.

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const session = request.cookies.get('session')
  const role = request.cookies.get('role')?.value

  // Block non-admin users from admin routes
  if (request.nextUrl.pathname.startsWith('/admin') && role !== 'admin') {
    return NextResponse.redirect(new URL('/unauthorized', request.url))
  }

  // Attach user info to request headers for downstream pages
  const requestHeaders = new Headers(request.headers)
  if (session?.value) {
    requestHeaders.set('x-session-id', session.value)
  }

  return NextResponse.next({
    request: { headers: requestHeaders },
  })
}
```

```tsx
// app/dashboard/page.tsx — read header set by middleware
import { headers } from 'next/headers'

export default async function DashboardPage() {
  const headersList = await headers()
  const sessionId = headersList.get('x-session-id')

  return <p>Session: {sessionId ?? 'Guest'}</p>
}
```

| Source | Read in middleware | Set in middleware |
| ------ | ------------------ | ----------------- |
| Cookies | `request.cookies.get('name')` | `response.cookies.set('name', value)` |
| Headers | `request.headers.get('name')` | `NextResponse.next({ request: { headers } })` |

# 6. Setting Response Cookies

Middleware can set or clear cookies on the response.

```tsx
// middleware.ts — refresh session cookie on each visit
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const response = NextResponse.next()
  const session = request.cookies.get('session')

  if (session) {
    // Extend session expiry on each request
    response.cookies.set('session', session.value, {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'lax',
      maxAge: 60 * 60 * 24 * 7, // 7 days
      path: '/',
    })
  }

  return response
}
```

| Cookie option | Purpose |
| ------------- | ------- |
| `httpOnly: true` | Not accessible via JavaScript — safer |
| `secure: true` | HTTPS only in production |
| `sameSite: 'lax'` | CSRF protection |
| `maxAge` | Expiry in seconds |

# 7. URL Rewrites

Rewrites serve a different route internally without changing the browser URL.

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // Serve /docs/getting-started from /documentation/start
  if (pathname.startsWith('/docs')) {
    const newPath = pathname.replace('/docs', '/documentation')
    return NextResponse.rewrite(new URL(newPath, request.url))
  }

  return NextResponse.next()
}
```

| Redirect | Rewrite |
| -------- | ------- |
| Browser URL changes | Browser URL stays the same |
| User sees new URL | User sees original URL |
| Use for auth gates | Use for internal routing, i18n |

# 8. Locale Detection Example

Common middleware pattern for internationalization.

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

const LOCALES = ['en', 'hi', 'ta']
const DEFAULT_LOCALE = 'en'

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // Skip if path already has locale prefix
  const pathnameHasLocale = LOCALES.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  )

  if (pathnameHasLocale) return NextResponse.next()

  // Detect from Accept-Language header or cookie
  const cookieLocale = request.cookies.get('locale')?.value
  const acceptLanguage = request.headers.get('accept-language')?.split(',')[0]?.split('-')[0]
  const locale = cookieLocale ?? acceptLanguage ?? DEFAULT_LOCALE

  const validLocale = LOCALES.includes(locale) ? locale : DEFAULT_LOCALE

  return NextResponse.redirect(
    new URL(`/${validLocale}${pathname}`, request.url)
  )
}

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
}
```

# 9. Auth Check Pattern (Complete Example)

```tsx
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

interface SessionPayload {
  userId: string
  role: string
  exp: number
}

function parseSession(token: string): SessionPayload | null {
  try {
    const payload = JSON.parse(atob(token.split('.')[1]))
    if (payload.exp * 1000 < Date.now()) return null
    return payload
  } catch {
    return null
  }
}

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // Public routes
  if (['/login', '/register', '/'].includes(pathname)) {
    return NextResponse.next()
  }

  const token = request.cookies.get('session')?.value
  const session = token ? parseSession(token) : null

  if (!session) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  if (pathname.startsWith('/admin') && session.role !== 'admin') {
    return NextResponse.redirect(new URL('/unauthorized', request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/admin/:path*', '/profile/:path*'],
}
```

| Check | Where |
| ----- | ----- |
| Has valid session? | Middleware — block before page load |
| Has admin role? | Middleware — route-level role gate |
| Can edit this resource? | Server Component / Server Action — resource-level |

**Never rely on middleware alone for sensitive operations** — always re-verify in Server Actions and Route Handlers.

# 10. Middleware Limitations

| Limitation | Explanation |
| ---------- | ----------- |
| Edge Runtime | No Node.js `fs`, `crypto` (full), native DB drivers |
| No response body | Cannot return HTML — only redirect, rewrite, headers |
| Runs on every matched request | Keep logic fast — avoid heavy computation |
| Cannot access Server Components | Runs before React rendering |
| Database calls | Use edge-compatible DB (e.g. Prisma Data Proxy, HTTP API) |

```tsx
// ❌ Avoid — heavy DB query in middleware
export async function middleware(request: NextRequest) {
  const user = await db.user.findUnique({ ... }) // Slow, may not work on Edge
}

// ✅ Better — verify JWT from cookie, defer DB check to page/action
export function middleware(request: NextRequest) {
  const token = request.cookies.get('session')?.value
  if (!token || !isValidJwt(token)) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  return NextResponse.next()
}
```

# 11. Best Practices

✔ Keep middleware fast — it runs on every matched request

✔ Use `matcher` to limit scope — don't run on static assets

✔ Redirect unauthenticated users to login with `callbackUrl`

✔ Set cookies with `httpOnly`, `secure`, and `sameSite`

✔ Re-verify auth in Server Actions and Route Handlers — middleware is first gate only

✔ Use rewrites for i18n and internal routing — redirects for auth

✔ Avoid heavy DB calls — verify tokens, not full user records
