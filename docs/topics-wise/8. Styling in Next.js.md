# Styling in Next.js - Teaching Notes

## Topic 8: Global CSS, CSS Modules, Tailwind, and Dark Mode

---

# 1. Why This Topic Matters

Students already know CSS from React training. Next.js adds conventions for **where** styles live and **how** they interact with Server Components.

Wrong styling choices cause:

* Global class name collisions
* Bloated client bundles from CSS-in-JS in Server Components
* Inconsistent responsive design across routes
* Flash of wrong theme on page load

This topic covers all primary styling approaches in Next.js 15 with App Router and TypeScript.

---

# 2. Styling Options Overview

| Method | File example | Scope | Server Component safe? |
| ------ | ------------ | ----- | ---------------------- |
| Global CSS | `app/globals.css` | Entire app | ✅ Yes |
| CSS Modules | `Button.module.css` | One component | ✅ Yes |
| Tailwind CSS | className in JSX | Utility classes | ✅ Yes |
| CSS-in-JS (styled-components) | JS defined styles | Component | ❌ Requires `'use client'` |
| Inline styles | `style={{}}` | Element | ✅ Yes (limited use) |

**Recommended stack for this course:** Global CSS + Tailwind CSS + CSS Modules for special cases.

# 3. Global CSS in app/globals.css

Global styles apply to the entire application. Import once in the **root layout**.

```css
/* app/globals.css */
*,
*::before,
*::after {
  box-sizing: border-box;
}

body {
  margin: 0;
  font-family: system-ui, sans-serif;
  line-height: 1.5;
  color: #1a1a1a;
  background-color: #ffffff;
}

a {
  color: inherit;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}
```

```tsx
// app/layout.tsx
import './globals.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

| Global CSS use case | Example |
| ------------------- | ------- |
| CSS reset / normalize | Box-sizing, margin zero |
| Base typography | `body` font, line-height |
| CSS variables | Theme colors (see dark mode) |
| Tailwind directives | `@tailwind base/components/utilities` |

Compare with react-project-study:

```tsx
// react-project-study/src/main.tsx
import './index.css'  // global styles entry
```

Same concept — one global import in the root entry (layout in Next.js).

### Rules for global CSS

| Rule | Detail |
| ---- | ------ |
| Import only in root layout | Importing global CSS elsewhere causes duplicate styles |
| Avoid component-specific rules | Use CSS Modules or Tailwind for component styles |
| Use CSS variables for theming | Enables dark mode without duplicating rules |

# 4. CSS Modules in Components

**CSS Modules** scope class names to a single component file — no collisions.

```css
/* components/ui/Card.module.css */
.card {
  padding: 16px;
  border: 1px solid #e5e7eb;
  border-radius: 8px;
  background: #ffffff;
}

.title {
  font-size: 1.125rem;
  font-weight: 600;
  margin-bottom: 8px;
}

.highlight {
  border-color: #3b82f6;
  box-shadow: 0 0 0 2px rgba(59, 130, 246, 0.2);
}
```

```tsx
// components/ui/Card.tsx
import styles from './Card.module.css'

interface CardProps {
  title: string
  children: React.ReactNode
  isHighlighted?: boolean
}

export function Card({ title, children, isHighlighted = false }: CardProps) {
  const cardClass = isHighlighted
    ? `${styles.card} ${styles.highlight}`
    : styles.card

  return (
    <article className={cardClass}>
      <h3 className={styles.title}>{title}</h3>
      {children}
    </article>
  )
}
```

| CSS Modules feature | Detail |
| ------------------- | ------ |
| Scoped class names | `.card` becomes `.Card_card_x7f2a` at build time |
| Import as object | `styles.card` — TypeScript knows class names |
| Works in Server Components | ✅ No client JS needed |
| File naming | `ComponentName.module.css` next to component |

Compare with react-project-study:

```tsx
// react-project-study/src/page2/Home/home.css — regular CSS file (global if imported)
import './home.css'
```

CSS Modules prevent accidental global leakage.

# 5. Tailwind CSS Setup with Next.js

create-next-app can configure Tailwind automatically. Key files:

```css
/* app/globals.css — with Tailwind */
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  body {
    @apply antialiased text-gray-900 bg-white;
  }
}
```

```typescript
// tailwind.config.ts
import type { Config } from 'tailwindcss'

const config: Config = {
  content: [
    './app/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  theme: {
    extend: {
      colors: {
        brand: {
          500: '#3b82f6',
          600: '#2563eb',
        },
      },
    },
  },
  plugins: [],
}

export default config
```

### Using Tailwind in components

```tsx
// app/products/page.tsx — Server Component
export default async function ProductsPage() {
  return (
    <main className="max-w-6xl mx-auto px-4 py-8">
      <h1 className="text-3xl font-bold text-gray-900 mb-6">
        Products
      </h1>
      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6">
        {/* product cards */}
      </div>
    </main>
  )
}
```

| Tailwind benefit in Next.js | Detail |
| --------------------------- | ------ |
| No runtime CSS-in-JS | Styles extracted at build — Server Component safe |
| Purge unused classes | Production bundle only includes used utilities |
| Responsive prefixes | `sm:`, `md:`, `lg:` for mobile-first design |
| Works everywhere | Server and Client Components |

# 6. Conditional Class Names

Components often need dynamic classes based on props or state.

### Template literal approach

```tsx
interface BadgeProps {
  label: string
  variant: 'success' | 'warning' | 'error'
}

const variantClasses = {
  success: 'bg-green-100 text-green-800',
  warning: 'bg-yellow-100 text-yellow-800',
  error: 'bg-red-100 text-red-800',
}

export function Badge({ label, variant }: BadgeProps) {
  return (
    <span
      className={`inline-block px-2 py-1 rounded text-sm font-medium ${variantClasses[variant]}`}
    >
      {label}
    </span>
  )
}
```

### clsx / classnames library (recommended)

```bash
npm install clsx
```

```tsx
import clsx from 'clsx'

interface ButtonProps {
  label: string
  isLoading?: boolean
  isDisabled?: boolean
}

export function Button({ label, isLoading, isDisabled }: ButtonProps) {
  return (
    <button
      disabled={isDisabled || isLoading}
      className={clsx(
        'px-4 py-2 rounded font-medium transition-colors',
        'bg-brand-500 text-white hover:bg-brand-600',
        isDisabled && 'opacity-50 cursor-not-allowed',
        isLoading && 'animate-pulse',
      )}
    >
      {isLoading ? 'Loading...' : label}
    </button>
  )
}
```

| Pattern | When to use |
| ------- | ----------- |
| Template literal | 2–3 simple conditions |
| `clsx()` | Multiple conditional classes |
| Variant map object | Fixed set of style variants |
| CSS Modules + clsx | Scoped styles with conditions |

### Client Component conditional example

```tsx
'use client'

import { useState } from 'react'
import clsx from 'clsx'

export function TabNav() {
  const [activeTab, setActiveTab] = useState('overview')

  return (
    <nav className="flex gap-4 border-b">
      {['overview', 'settings', 'billing'].map((tab) => (
        <button
          key={tab}
          onClick={() => setActiveTab(tab)}
          className={clsx(
            'pb-2 capitalize',
            activeTab === tab
              ? 'border-b-2 border-brand-500 text-brand-600 font-semibold'
              : 'text-gray-500 hover:text-gray-700',
          )}
        >
          {tab}
        </button>
      ))}
    </nav>
  )
}
```

# 7. Responsive Design with Tailwind

Tailwind uses a **mobile-first** breakpoint system.

| Prefix | Min width | Typical use |
| ------ | --------- | ----------- |
| (none) | 0px | Mobile default |
| `sm:` | 640px | Large phones |
| `md:` | 768px | Tablets |
| `lg:` | 1024px | Laptops |
| `xl:` | 1280px | Desktops |
| `2xl:` | 1536px | Large screens |

```tsx
// Responsive product grid
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4">
  {products.map((p) => (
    <ProductCard key={p.id} product={p} />
  ))}
</div>
```

```tsx
// Responsive navigation
<header className="flex flex-col md:flex-row md:items-center md:justify-between gap-4 p-4">
  <h1 className="text-xl md:text-2xl font-bold">Training Portal</h1>
  <nav className="flex flex-wrap gap-2 md:gap-4">
    <Link href="/">Home</Link>
    <Link href="/products">Products</Link>
  </nav>
</header>
```

| Layout pattern | Tailwind classes |
| -------------- | ---------------- |
| Centered container | `max-w-6xl mx-auto px-4` |
| Flex row on desktop, column on mobile | `flex flex-col md:flex-row` |
| Hide on mobile | `hidden md:block` |
| Full width mobile, fixed sidebar desktop | `w-full lg:w-64` |

Compare with react-project-study CSS:

```css
/* react-project-study — custom CSS for responsive */
@media (max-width: 768px) {
  .product-list { flex-direction: column; }
}
```

Tailwind replaces many media queries with utility prefixes.

# 8. CSS-in-JS Considerations with Server Components

Libraries like **styled-components** and **Emotion** require runtime JavaScript — they need **`'use client'`**.

```tsx
// ❌ styled-components in Server Component — not supported
import styled from 'styled-components'

const Title = styled.h1`
  color: blue;
`
```

```tsx
// ✅ Alternative — Tailwind or CSS Modules in Server Component
export function PageTitle({ children }: { children: React.ReactNode }) {
  return <h1 className="text-2xl font-bold text-blue-600">{children}</h1>
}
```

| Approach | Server Component | Client Component |
| -------- | ---------------- | ---------------- |
| Tailwind | ✅ Preferred | ✅ |
| CSS Modules | ✅ Preferred | ✅ |
| Global CSS | ✅ | ✅ |
| styled-components | ❌ | ✅ (with registry setup) |
| Emotion | ❌ | ✅ |
| Inline `style={{}}` | ✅ (limited) | ✅ |

**Recommendation:** Use Tailwind + CSS Modules. Avoid CSS-in-JS unless the team already standardized on it.

# 9. Dark Mode Basics

Dark mode can be implemented with CSS variables, Tailwind's `dark:` variant, and a client-side theme toggle.

### Step 1 — CSS variables in globals.css

```css
/* app/globals.css */
:root {
  --background: #ffffff;
  --foreground: #1a1a1a;
  --card: #f9fafb;
  --border: #e5e7eb;
}

.dark {
  --background: #0f172a;
  --foreground: #f1f5f9;
  --card: #1e293b;
  --border: #334155;
}

body {
  background-color: var(--background);
  color: var(--foreground);
}
```

### Step 2 — Tailwind dark mode config

```typescript
// tailwind.config.ts
const config: Config = {
  darkMode: 'class', // toggle via class on <html>
  // ...
}
```

### Step 3 — Use dark: prefix

```tsx
<div className="bg-white dark:bg-slate-900 text-gray-900 dark:text-gray-100 border border-gray-200 dark:border-slate-700 rounded-lg p-4">
  <h2 className="text-lg font-semibold">Card title</h2>
  <p className="text-gray-600 dark:text-gray-400">Card content</p>
</div>
```

### Step 4 — Theme toggle (Client Component)

```tsx
// components/ThemeToggle.tsx
'use client'

import { useEffect, useState } from 'react'

export function ThemeToggle() {
  const [isDark, setIsDark] = useState(false)

  useEffect(() => {
    const root = document.documentElement
    if (isDark) {
      root.classList.add('dark')
    } else {
      root.classList.remove('dark')
    }
  }, [isDark])

  return (
    <button
      onClick={() => setIsDark(!isDark)}
      aria-label={isDark ? 'Switch to light mode' : 'Switch to dark mode'}
      className="p-2 rounded border"
    >
      {isDark ? '☀️' : '🌙'}
    </button>
  )
}
```

| Dark mode strategy | Detail |
| ------------------ | ------ |
| `darkMode: 'class'` | Add/remove `dark` class on `<html>` |
| `darkMode: 'media'` | Follows system preference — no toggle needed |
| CSS variables | Shared tokens for custom properties |
| `dark:` prefix | Tailwind utilities for dark variant |

For production apps, use **`next-themes`** library to avoid flash of wrong theme — covered in intermediate topics.

# 12. Best Practices

✔ Import **global CSS only in root layout**

✔ Use **Tailwind** for most styling — fast, Server Component safe

✔ Use **CSS Modules** for complex component-specific styles

✔ Use **`clsx`** for conditional class names

✔ Design **mobile-first** with Tailwind responsive prefixes

✔ Avoid **CSS-in-JS** in Server Components

✔ Define **CSS variables** for theme tokens (colors, spacing)

✔ Keep **layout styles** in `layout.tsx`, not duplicated in pages

✔ Test dark mode on both themes before shipping
