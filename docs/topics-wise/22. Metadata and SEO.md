# Metadata and SEO - Teaching Notes

## Topic 22: Titles, Open Graph, Sitemaps, and Search Optimization

---

# 1. Why This Topic Matters

Search engines and social platforms read your page's metadata — not just visible content. Without proper titles, descriptions, and Open Graph tags, your pages look broken when shared on WhatsApp, Twitter, or LinkedIn, and rank poorly in Google.

Next.js App Router provides a powerful Metadata API for static and dynamic metadata, plus file conventions for `robots.txt` and sitemaps. This topic covers everything needed for professional SEO.

---

# 2. The Metadata API Overview

Export a `metadata` object or `generateMetadata` function from any `layout.tsx` or `page.tsx`.

```tsx
// app/layout.tsx — static metadata for the whole app
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'My Training App',
    template: '%s | My Training App',
  },
  description: 'Learn Next.js from beginner to advanced with hands-on projects.',
  metadataBase: new URL('https://mytrainingapp.com'),
}

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

| Export | Type | Use |
| ------ | ---- | --- |
| `metadata` | Static object | Fixed title, description |
| `generateMetadata` | Async function | Dynamic title from fetched data |
| `metadataBase` | URL | Resolves relative URLs in metadata |

The `template` pattern: child pages set `title: 'Products'` → renders as `"Products | My Training App"`.

# 3. Static Metadata on Pages

```tsx
// app/about/page.tsx
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'About Us',
  description: 'Learn about our mission to teach modern web development.',
  keywords: ['Next.js training', 'React course', 'web development'],
  authors: [{ name: 'Training Team' }],
  openGraph: {
    title: 'About Us — My Training App',
    description: 'Learn about our mission to teach modern web development.',
    url: 'https://mytrainingapp.com/about',
    siteName: 'My Training App',
    locale: 'en_IN',
    type: 'website',
  },
  twitter: {
    card: 'summary_large_image',
    title: 'About Us — My Training App',
    description: 'Learn about our mission to teach modern web development.',
  },
}

export default function AboutPage() {
  return (
    <main>
      <h1>About Us</h1>
      <p>We teach Next.js and React to developers across India.</p>
    </main>
  )
}
```

| Field | Purpose |
| ----- | ------- |
| `title` | Browser tab + search result heading |
| `description` | Search result snippet (150–160 chars ideal) |
| `keywords` | Low SEO value today — optional |
| `openGraph` | Facebook, WhatsApp, LinkedIn previews |
| `twitter` | Twitter/X card preview |

# 4. generateMetadata for Dynamic Pages

Fetch data and generate metadata per route — essential for blog posts and product pages.

```tsx
// app/blog/[slug]/page.tsx
import type { Metadata } from 'next'
import { getPostBySlug } from '@/lib/posts'
import { notFound } from 'next/navigation'

interface BlogPostPageProps {
  params: Promise<{ slug: string }>
}

export async function generateMetadata({ params }: BlogPostPageProps): Promise<Metadata> {
  const { slug } = await params
  const post = await getPostBySlug(slug)

  if (!post) {
    return { title: 'Post Not Found' }
  }

  return {
    title: post.title,
    description: post.excerpt,
    authors: [{ name: post.author }],
    openGraph: {
      title: post.title,
      description: post.excerpt,
      type: 'article',
      publishedTime: post.publishedAt,
      authors: [post.author],
      images: [
        {
          url: post.coverImage,
          width: 1200,
          height: 630,
          alt: post.title,
        },
      ],
    },
    twitter: {
      card: 'summary_large_image',
      title: post.title,
      description: post.excerpt,
      images: [post.coverImage],
    },
    alternates: {
      canonical: `https://mytrainingapp.com/blog/${slug}`,
    },
  }
}

export default async function BlogPostPage({ params }: BlogPostPageProps) {
  const { slug } = await params
  const post = await getPostBySlug(slug)

  if (!post) notFound()

  return (
    <article>
      <h1>{post.title}</h1>
      <time dateTime={post.publishedAt}>
        {new Date(post.publishedAt).toLocaleDateString('en-IN')}
      </time>
      <div>{post.content}</div>
    </article>
  )
}
```

| Static `metadata` | `generateMetadata` |
| ----------------- | ------------------ |
| Fixed values | Values from fetched data |
| Sync export | Async function |
| `/about`, `/contact` | `/blog/[slug]`, `/products/[id]` |

# 5. Open Graph and Social Sharing

Open Graph tags control how links appear when shared on social platforms.

```tsx
export const metadata: Metadata = {
  openGraph: {
    title: 'Next.js Training Course',
    description: 'Master Next.js from beginner to production.',
    url: 'https://mytrainingapp.com',
    siteName: 'My Training App',
    images: [
      {
        url: '/og-image.png',       // resolved against metadataBase
        width: 1200,
        height: 630,
        alt: 'Next.js Training Course',
      },
    ],
    locale: 'en_IN',
    type: 'website',
  },
}
```

| OG property | What it controls |
| ----------- | ---------------- |
| `og:title` | Card title on WhatsApp, LinkedIn |
| `og:description` | Card description text |
| `og:image` | Preview image (1200×630 recommended) |
| `og:url` | Canonical URL for the shared link |
| `og:type` | `website`, `article`, `product` |

**Image requirements:**
* Minimum: 1200×630 pixels
* Format: PNG or JPG
* Place in `public/og-image.png`
* Set `metadataBase` so relative paths resolve correctly

# 6. Favicon and App Icons

Next.js supports file-based metadata for icons — no code needed.

```text
app/
  favicon.ico           → /favicon.ico
  icon.png              → app icon
  icon.svg              → scalable app icon
  apple-icon.png        → Apple touch icon
  opengraph-image.png   → default OG image for this route
  twitter-image.png     → default Twitter card image
```

```tsx
// Or define icons in metadata export
export const metadata: Metadata = {
  icons: {
    icon: '/favicon.ico',
    shortcut: '/shortcut-icon.png',
    apple: '/apple-icon.png',
  },
}
```

| File | Purpose |
| ---- | ------- |
| `favicon.ico` | Browser tab icon |
| `icon.png` / `icon.svg` | PWA and mobile icon |
| `apple-icon.png` | iOS home screen icon |
| `opengraph-image.png` | Auto OG image for route segment |

# 7. robots.txt

Control which pages search engines can crawl.

```tsx
// app/robots.ts
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      {
        userAgent: '*',
        allow: '/',
        disallow: ['/admin/', '/api/', '/dashboard/'],
      },
    ],
    sitemap: 'https://mytrainingapp.com/sitemap.xml',
  }
}
```

| Directive | Meaning |
| --------- | ------- |
| `allow: '/'` | Crawl all public pages |
| `disallow: '/admin/'` | Block admin from indexing |
| `sitemap:` | Points crawlers to your sitemap |

This generates `/robots.txt` automatically at build time.

# 8. sitemap.ts

Help search engines discover all your pages.

```tsx
// app/sitemap.ts
import type { MetadataRoute } from 'next'
import { getAllPosts } from '@/lib/posts'
import { getAllProducts } from '@/lib/products'

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const baseUrl = 'https://mytrainingapp.com'

  const staticPages: MetadataRoute.Sitemap = [
    { url: baseUrl, lastModified: new Date(), changeFrequency: 'weekly', priority: 1 },
    { url: `${baseUrl}/about`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.8 },
    { url: `${baseUrl}/products`, lastModified: new Date(), changeFrequency: 'daily', priority: 0.9 },
  ]

  const posts = await getAllPosts()
  const postPages: MetadataRoute.Sitemap = posts.map((post) => ({
    url: `${baseUrl}/blog/${post.slug}`,
    lastModified: new Date(post.updatedAt),
    changeFrequency: 'weekly',
    priority: 0.7,
  }))

  const products = await getAllProducts()
  const productPages: MetadataRoute.Sitemap = products.map((product) => ({
    url: `${baseUrl}/products/${product.id}`,
    lastModified: new Date(product.updatedAt),
    changeFrequency: 'daily',
    priority: 0.8,
  }))

  return [...staticPages, ...postPages, ...productPages]
}
```

| Field | Purpose |
| ----- | ------- |
| `url` | Full page URL |
| `lastModified` | When page content last changed |
| `changeFrequency` | How often content updates |
| `priority` | Relative importance (0.0–1.0) |

Generates `/sitemap.xml` automatically.

# 9. Canonical URLs

Prevent duplicate content issues when the same page is accessible at multiple URLs.

```tsx
// app/products/page.tsx
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Products',
  alternates: {
    canonical: 'https://mytrainingapp.com/products',
  },
}
```

```tsx
// Dynamic page — canonical includes slug
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { slug } = await params
  return {
    alternates: {
      canonical: `https://mytrainingapp.com/blog/${slug}`,
    },
  }
}
```

| Problem | Solution |
| ------- | -------- |
| `/products?page=2` indexed separately | Canonical points to `/products` |
| Trailing slash duplicates | Consistent URL in canonical |
| HTTP vs HTTPS duplicates | Always use HTTPS in canonical |

# 10. SEO Best Practices for Next.js

### Semantic HTML helps SEO

```tsx
// ✅ Good — semantic structure
<main>
  <article>
    <header>
      <h1>{post.title}</h1>
      <time dateTime={post.publishedAt}>{formattedDate}</time>
    </header>
    <section>{post.content}</section>
  </article>
</main>

// ❌ Avoid — div soup with no semantic meaning
<div>
  <div><div>{post.title}</div></div>
</div>
```

### Structured data (JSON-LD)

```tsx
// app/products/[id]/page.tsx
export default async function ProductPage({ params }: Props) {
  const { id } = await params
  const product = await getProductById(id)

  const jsonLd = {
    '@context': 'https://schema.org',
    '@type': 'Product',
    name: product.name,
    description: product.description,
    offers: {
      '@type': 'Offer',
      price: product.price,
      priceCurrency: 'INR',
    },
  }

  return (
    <main>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      <h1>{product.name}</h1>
      <p>Rs. {product.price}</p>
    </main>
  )
}
```

| SEO factor | Next.js feature |
| ---------- | --------------- |
| Page title | `metadata.title` |
| Description | `metadata.description` |
| Social previews | `openGraph`, `twitter` |
| Crawlability | `robots.ts`, `sitemap.ts` |
| Duplicate content | `alternates.canonical` |
| Rich results | JSON-LD structured data |
| Performance (Core Web Vitals) | Server Components, streaming |

# 11. Metadata in Layouts

Layouts can export metadata inherited by all child pages.

```tsx
// app/blog/layout.tsx
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Blog',
    template: '%s | Blog | My Training App',
  },
  description: 'Articles about Next.js, React, and modern web development.',
}

export default function BlogLayout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <nav aria-label="Blog navigation">
        <a href="/blog">All Posts</a>
      </nav>
      {children}
    </div>
  )
}
```

Child page sets `title: 'My First Post'` → renders as `"My First Post | Blog | My Training App"`.

# 12. Best Practices

✔ Set `metadataBase` in root layout for correct relative URL resolution

✔ Use `generateMetadata` for all dynamic routes (blog, products, profiles)

✔ Write unique titles and descriptions for every page — no duplicates

✔ Add Open Graph images (1200×630) for all shareable pages

✔ Generate `sitemap.ts` and `robots.ts` for every production app

✔ Set canonical URLs on pages with query param variants

✔ Use semantic HTML — metadata alone is not enough for SEO

✔ Keep descriptions between 150–160 characters
