# Server Actions - Teaching Notes

## Topic 23: Server-Side Mutations with 'use server'

---

# 1. Why This Topic Matters

Server Actions are async functions that run **on the server** but can be called from forms and Client Components. They replace much of the boilerplate of API routes for mutations — create, update, delete — inside your Next.js app.

With Server Actions, forms work **without JavaScript** (progressive enhancement), validation runs on the server where it can't be bypassed, and you can revalidate cached data after mutations. This is one of the most powerful features of the App Router.

---

# 2. What Are Server Actions?

Server Actions are functions marked with `'use server'` that execute on the server when called from the client or a form.

```text
Browser form submit
       │
       ▼
Server Action ('use server')
       │
       ├── Validate input
       ├── Write to database
       ├── revalidatePath / revalidateTag
       │
       ▼
Return result to client (or redirect)
```

| Feature | Detail |
| ------- | ------ |
| Directive | `'use server'` at top of file or function |
| Called from | `<form action={...}>`, Client Components |
| Runs on | Server only |
| Progressive enhancement | Forms work without JavaScript |
| HTTP method | POST (automatic) |

---

# 3. Inline Server Actions

Define actions directly in a Server Component file.

```tsx
// app/contact/page.tsx
import { revalidatePath } from 'next/cache'

async function submitContact(formData: FormData) {
  'use server'

  const name = formData.get('name') as string
  const email = formData.get('email') as string
  const message = formData.get('message') as string

  if (!name?.trim() || !email?.trim() || !message?.trim()) {
    return { error: 'All fields are required.' }
  }

  await db.contact.create({ data: { name, email, message } })
  revalidatePath('/contact')

  return { success: true }
}

export default function ContactPage() {
  return (
    <main>
      <h1>Contact Us</h1>
      <form action={submitContact}>
        <input name="name" placeholder="Your name" required />
        <input name="email" type="email" placeholder="Email" required />
        <textarea name="message" placeholder="Message" required />
        <button type="submit">Send Message</button>
      </form>
    </main>
  )
}
```

| Pattern | When to use |
| ------- | ----------- |
| Inline `'use server'` | Simple, page-specific actions |
| Separate file | Reusable actions across pages |
| Bound to `<form action>` | Progressive enhancement — no JS needed |

# 4. Server Actions in Separate Files

Recommended for reusable actions — keeps components clean.

```tsx
// app/actions/tasks.ts
'use server'

import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'
import { db } from '@/lib/db'

export interface TaskActionResult {
  success: boolean
  error?: string
}

export async function createTask(formData: FormData): Promise<TaskActionResult> {
  const title = formData.get('title') as string

  if (!title?.trim()) {
    return { success: false, error: 'Task title is required.' }
  }

  if (title.length > 200) {
    return { success: false, error: 'Title must be under 200 characters.' }
  }

  try {
    await db.task.create({ data: { title: title.trim(), completed: false } })
    revalidatePath('/tasks')
    return { success: true }
  } catch (error) {
    console.error('createTask failed:', error)
    return { success: false, error: 'Could not create task. Please try again.' }
  }
}

export async function deleteTask(taskId: string): Promise<TaskActionResult> {
  if (!taskId) {
    return { success: false, error: 'Task ID is required.' }
  }

  try {
    await db.task.delete({ where: { id: taskId } })
    revalidatePath('/tasks')
    return { success: true }
  } catch (error) {
    console.error('deleteTask failed:', error)
    return { success: false, error: 'Could not delete task.' }
  }
}

export async function toggleTask(taskId: string, completed: boolean): Promise<void> {
  await db.task.update({ where: { id: taskId }, data: { completed } })
  revalidatePath('/tasks')
}
```

```tsx
// app/tasks/page.tsx
import { createTask, deleteTask } from '@/app/actions/tasks'
import { TaskList } from '@/components/task-list'
import { getTasks } from '@/lib/tasks'

export default async function TasksPage() {
  const tasks = await getTasks()

  return (
    <main>
      <h1>Tasks</h1>

      <form action={createTask}>
        <input name="title" placeholder="New task..." required />
        <button type="submit">Add Task</button>
      </form>

      <TaskList tasks={tasks} deleteAction={deleteTask} />
    </main>
  )
}
```

# 5. Form Actions with useActionState

`useActionState` (React 19) handles action results, pending state, and errors in Client Components.

```tsx
// components/create-task-form.tsx
'use client'

import { useActionState } from 'react'
import { createTask, type TaskActionResult } from '@/app/actions/tasks'

const initialState: TaskActionResult = { success: false }

export function CreateTaskForm() {
  const [state, action, isPending] = useActionState(createTask, initialState)

  return (
    <form action={action}>
      {state.error && (
        <p role="alert" className="text-red-600">{state.error}</p>
      )}
      {state.success && (
        <p role="status" className="text-green-600">Task created!</p>
      )}

      <input
        name="title"
        placeholder="What needs to be done?"
        disabled={isPending}
        required
      />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Adding...' : 'Add Task'}
      </button>
    </form>
  )
}
```

| Hook / API | Purpose |
| ---------- | ------- |
| `useActionState(action, initialState)` | Track result + pending state |
| `isPending` | True while action is running — disable form |
| `state.error` | Validation or server error message |
| `state.success` | Show success feedback |

# 6. Calling Server Actions from Client Components

Pass actions as props or import directly in Client Components.

```tsx
// components/delete-task-button.tsx
'use client'

import { useTransition } from 'react'
import { deleteTask } from '@/app/actions/tasks'

interface DeleteTaskButtonProps {
  taskId: string
  taskTitle: string
}

export function DeleteTaskButton({ taskId, taskTitle }: DeleteTaskButtonProps) {
  const [isPending, startTransition] = useTransition()

  function handleDelete() {
    if (!confirm(`Delete "${taskTitle}"?`)) return

    startTransition(async () => {
      const result = await deleteTask(taskId)
      if (!result.success) {
        alert(result.error ?? 'Delete failed')
      }
    })
  }

  return (
    <button
      type="button"
      onClick={handleDelete}
      disabled={isPending}
      aria-label={`Delete task: ${taskTitle}`}
    >
      {isPending ? 'Deleting...' : 'Delete'}
    </button>
  )
}
```

| Pattern | Use |
| ------- | --- |
| `<form action={action}>` | Form submit — progressive enhancement |
| `useActionState` | Form with result/error display |
| `useTransition` + direct call | Button click without form |
| Pass action as prop | Server Component passes to Client Component |

# 7. Validating Input on the Server

Never trust client-side validation alone — always validate in Server Actions.

```tsx
// app/actions/register.ts
'use server'

import { z } from 'zod'

const registerSchema = z.object({
  name: z.string().min(2, 'Name must be at least 2 characters'),
  email: z.string().email('Invalid email address'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
})

export interface RegisterResult {
  success: boolean
  errors?: Record<string, string[]>
}

export async function registerUser(formData: FormData): Promise<RegisterResult> {
  const raw = {
    name: formData.get('name'),
    email: formData.get('email'),
    password: formData.get('password'),
  }

  const parsed = registerSchema.safeParse(raw)

  if (!parsed.success) {
    return {
      success: false,
      errors: parsed.error.flatten().fieldErrors,
    }
  }

  const { name, email, password } = parsed.data

  const existing = await db.user.findUnique({ where: { email } })
  if (existing) {
    return { success: false, errors: { email: ['Email already registered'] } }
  }

  const hashedPassword = await hashPassword(password)
  await db.user.create({ data: { name, email, password: hashedPassword } })

  return { success: true }
}
```

| Validation layer | Purpose |
| ---------------- | ------- |
| HTML `required`, `type="email"` | UX — instant feedback |
| Client-side (Zod + react-hook-form) | UX — before submit |
| Server Action (Zod) | Security — cannot be bypassed |

# 8. Revalidating After Mutations

After a Server Action changes data, tell Next.js to refresh cached pages.

```tsx
'use server'

import { revalidatePath, revalidateTag } from 'next/cache'

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string
  await db.post.create({ data: { title } })

  // Revalidate specific path
  revalidatePath('/blog')
  revalidatePath('/')

  return { success: true }
}

export async function updatePost(postId: string, formData: FormData) {
  const title = formData.get('title') as string
  await db.post.update({ where: { id: postId }, data: { title } })

  // Revalidate by cache tag
  revalidateTag('posts')
  revalidateTag(`post-${postId}`)

  return { success: true }
}
```

| Function | Scope |
| -------- | ----- |
| `revalidatePath('/blog')` | Re-render `/blog` on next visit |
| `revalidatePath('/blog', 'layout')` | Re-render layout + all nested pages |
| `revalidateTag('posts')` | Revalidate all fetches tagged `'posts'` |

# 9. Redirect After Action

Use `redirect()` from `next/navigation` in Server Actions after successful mutations.

```tsx
'use server'

import { redirect } from 'next/navigation'
import { revalidatePath } from 'next/cache'

export async function createProduct(formData: FormData) {
  const name = formData.get('name') as string
  const price = Number(formData.get('price'))

  if (!name?.trim() || isNaN(price) || price <= 0) {
    return { error: 'Invalid product data' }
  }

  const product = await db.product.create({
    data: { name: name.trim(), price },
  })

  revalidatePath('/products')
  redirect(`/products/${product.id}`) // throws — must be called outside try/catch
}
```

**Important:** `redirect()` throws internally — do not wrap it in `try/catch`.

```tsx
// ❌ Wrong — redirect caught by catch block
try {
  await db.product.create({ data })
  redirect('/products')
} catch (error) {
  return { error: 'Failed' }
}

// ✅ Correct — redirect outside try/catch
let product
try {
  product = await db.product.create({ data })
} catch (error) {
  return { error: 'Failed' }
}
redirect(`/products/${product.id}`)
```

# 10. Security Considerations

Server Actions are public endpoints — treat them like API routes.

```tsx
'use server'

import { auth } from '@/lib/auth'

export async function deletePost(postId: string) {
  // 1. Authenticate
  const session = await auth()
  if (!session?.user) {
    return { success: false, error: 'Unauthorized' }
  }

  // 2. Authorize — check ownership
  const post = await db.post.findUnique({ where: { id: postId } })
  if (!post) {
    return { success: false, error: 'Post not found' }
  }

  if (post.authorId !== session.user.id && session.user.role !== 'admin') {
    return { success: false, error: 'Forbidden' }
  }

  // 3. Validate input
  if (!postId || typeof postId !== 'string') {
    return { success: false, error: 'Invalid post ID' }
  }

  // 4. Perform mutation
  await db.post.delete({ where: { id: postId } })
  revalidatePath('/blog')

  return { success: true }
}
```

| Security check | Why |
| -------------- | --- |
| Authentication | Verify user is logged in |
| Authorization | Verify user owns the resource |
| Input validation | Prevent malformed data |
| Rate limiting | Prevent abuse (use middleware or library) |
| Never trust client | All checks must run on server |

# 11. Server Actions vs Route Handlers

| Scenario | Use |
| -------- | --- |
| Form in your Next.js app | Server Action |
| Delete button with confirmation | Server Action |
| Mobile app REST API | Route Handler |
| Webhook from Stripe/GitHub | Route Handler |
| Third-party API integration | Route Handler |
| Progressive enhancement form | Server Action |

---

# 12. Best Practices

✔ Define reusable actions in `app/actions/` files with `'use server'`

✔ Always validate input on the server with Zod or similar

✔ Return `{ success, error }` objects — don't throw for expected errors

✔ Call `revalidatePath` or `revalidateTag` after mutations

✔ Authenticate and authorize in every sensitive action

✔ Use `useActionState` for forms with error/success feedback

✔ Use `redirect()` after create — outside try/catch blocks
