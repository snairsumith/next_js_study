# Search Params and URL State - Teaching Notes

## Topic 17: Query Strings, Filtering, and Bookmarkable UI State

---

# 1. Why This Topic Matters

Search params (query strings) turn URL state into a powerful tool for filtering, sorting, pagination, and tabs. Unlike React state (`useState`), URL state is **shareable**, **bookmarkable**, and **survives page refresh**.

In Next.js App Router, you read search params on the **server** via the `searchParams` prop and on the **client** with `useSearchParams`. Mastering both approaches builds apps that feel professional and work without JavaScript.

---

# 2. What Are Search Params?

Everything after `?` in a URL is the query string.

```text
/products?category=electronics&sort=price&page=2
           â””â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€ search params
```

| URL | Meaning |
| --- | ------- |
| `?q=keyboard` | Search for "keyboard" |
| `?sort=price&order=asc` | Sort by price ascending |
| `?page=3&limit=20` | Pagination â€” page 3, 20 items |
| `?tab=reviews` | Active tab is reviews |
| `?category=phones&brand=samsung` | Multiple filters combined |

Search params are always **strings** (or arrays of strings). Parse numbers and booleans yourself.

# 3. Reading searchParams on the Server

Page components receive `searchParams` as a prop â€” similar to `params`.

```tsx
// app/products/page.tsx
interface ProductsPageProps {
  searchParams: Promise<{
    category?: string
    sort?: string
    page?: string
  }>
}

export default async function ProductsPage({ searchParams }: ProductsPageProps) {
  const { category, sort = 'name', page = '1' } = await searchParams

  const currentPage = Math.max(1, parseInt(page, 10) || 1)
  const limit = 12
  const offset = (currentPage - 1) * limit

  const products = await getProducts({
    category,
    sort,
    limit,
    offset,
  })

  return (
    <main>
      <h1>Products</h1>
      {category && <p>Filtered by: {category}</p>}
      <p>Sorted by: {sort} â€” Page {currentPage}</p>

      <ul>
        {products.map((product) => (
          <li key={product.id}>{product.name}</li>
        ))}
      </ul>
    </main>
  )
}
```

| Prop | Type (Next.js 15+) | Notes |
| ---- | ------------------ | ----- |
| `searchParams` | `Promise<Record<string, string \| string[]>>` | Always `await` first |
| Single value | `string` | `?sort=price` â†’ `"price"` |
| Repeated key | `string[]` | `?tag=a&tag=b` â†’ `["a", "b"]` |

**Note:** Like `params`, `searchParams` is a Promise in Next.js 15+.

# 4. Building Filter Links with Link

Use `<Link>` to update search params without full page reload (with prefetch).

```tsx
// components/product-filters.tsx â€” Server Component
import Link from 'next/link'

interface ProductFiltersProps {
  currentCategory?: string
  currentSort: string
}

const CATEGORIES = ['electronics', 'clothing', 'books', 'home']

export function ProductFilters({ currentCategory, currentSort }: ProductFiltersProps) {
  return (
    <nav aria-label="Product filters">
      <div>
        <h2>Category</h2>
        {CATEGORIES.map((category) => (
          <Link
            key={category}
            href={`/products?category=${category}&sort=${currentSort}`}
            aria-current={currentCategory === category ? 'page' : undefined}
          >
            {category}
          </Link>
        ))}
        {currentCategory && (
          <Link href={`/products?sort=${currentSort}`}>Clear filter</Link>
        )}
      </div>

      <div>
        <h2>Sort</h2>
        <Link href={`/products?sort=name${currentCategory ? `&category=${currentCategory}` : ''}`}>
          Name
        </Link>
        <Link href={`/products?sort=price${currentCategory ? `&category=${currentCategory}` : ''}`}>
          Price
        </Link>
      </div>
    </nav>
  )
}
```

| Approach | JS required? | SEO friendly? |
| -------- | ------------ | ------------- |
| `<Link href="?sort=price">` | No (progressive enhancement) | âś… Yes â€” server renders filtered page |
| `useRouter().push('?sort=price')` | Yes | âś… Yes â€” still navigates |
| `useState` for filter | Yes | âťŚ No â€” state lost on refresh |

# 5. useSearchParams on the Client

For interactive UI that reads or updates the URL without a full navigation, use the client hook.

```tsx
'use client'

import { useSearchParams, useRouter, usePathname } from 'next/navigation'
import { Suspense } from 'react'

function SearchBarInner() {
  const searchParams = useSearchParams()
  const router = useRouter()
  const pathname = usePathname()

  const currentQuery = searchParams.get('q') ?? ''

  function handleSearch(formData: FormData) {
    const query = formData.get('q') as string
    const params = new URLSearchParams(searchParams.toString())

    if (query.trim()) {
      params.set('q', query.trim())
    } else {
      params.delete('q')
    }

    params.delete('page') // reset pagination on new search
    router.push(`${pathname}?${params.toString()}`)
  }

  return (
    <form action={handleSearch}>
      <input
        type="search"
        name="q"
        defaultValue={currentQuery}
        placeholder="Search products..."
        aria-label="Search products"
      />
      <button type="submit">Search</button>
    </form>
  )
}

export function SearchBar() {
  return (
    <Suspense fallback={<input disabled placeholder="Loading..." />}>
      <SearchBarInner />
    </Suspense>
  )
}
```

| Hook | Purpose |
| ---- | ------- |
| `useSearchParams()` | Read current query string |
| `usePathname()` | Current path without query string |
| `useRouter()` | Programmatic navigation |

**Important:** Wrap components using `useSearchParams` in `<Suspense>` to avoid blocking static rendering during build.

# 6. Pagination with Search Params

Pagination is a classic search params use case.

```tsx
// components/pagination.tsx
import Link from 'next/link'

interface PaginationProps {
  currentPage: number
  totalPages: number
  baseParams: Record<string, string>
}

export function Pagination({ currentPage, totalPages, baseParams }: PaginationProps) {
  function buildPageUrl(page: number) {
    const params = new URLSearchParams({ ...baseParams, page: String(page) })
    return `?${params.toString()}`
  }

  return (
    <nav aria-label="Pagination">
      {currentPage > 1 && (
        <Link href={buildPageUrl(currentPage - 1)} aria-label="Previous page">
          Previous
        </Link>
      )}

      <span>
        Page {currentPage} of {totalPages}
      </span>

      {currentPage < totalPages && (
        <Link href={buildPageUrl(currentPage + 1)} aria-label="Next page">
          Next
        </Link>
      )}
    </nav>
  )
}
```

```tsx
// app/products/page.tsx (pagination section)
const { category, sort = 'name', page = '1' } = await searchParams
const currentPage = Math.max(1, parseInt(page, 10) || 1)
const { products, totalCount } = await getProducts({ category, sort, page: currentPage })
const totalPages = Math.ceil(totalCount / 12)

return (
  <>
    {/* product list */}
    <Pagination
      currentPage={currentPage}
      totalPages={totalPages}
      baseParams={{ sort, ...(category ? { category } : {}) }}
    />
  </>
)
```

| Param | Typical default | Validation |
| ----- | --------------- | ---------- |
| `page` | `1` | `Math.max(1, parseInt(page, 10) \|\| 1)` |
| `limit` | `12` or `20` | Cap at max (e.g. 100) |
| `sort` | `'name'` | Whitelist allowed values |

# 7. Sorting and Filtering Patterns

Always **whitelist** allowed sort/filter values â€” never trust raw URL input.

```tsx
// lib/search-params.ts
const ALLOWED_SORTS = ['name', 'price', 'createdAt'] as const
type SortOption = (typeof ALLOWED_SORTS)[number]

export function parseSortParam(raw?: string): SortOption {
  if (raw && ALLOWED_SORTS.includes(raw as SortOption)) {
    return raw as SortOption
  }
  return 'name'
}

export function parsePageParam(raw?: string): number {
  const parsed = parseInt(raw ?? '1', 10)
  return Number.isNaN(parsed) || parsed < 1 ? 1 : parsed
}
```

```tsx
// app/products/page.tsx
import { parseSortParam, parsePageParam } from '@/lib/search-params'

export default async function ProductsPage({ searchParams }: ProductsPageProps) {
  const params = await searchParams
  const sort = parseSortParam(params.sort)
  const page = parsePageParam(params.page)
  const category = typeof params.category === 'string' ? params.category : undefined

  const products = await getProducts({ sort, page, category })
  // ...
}
```

| Security concern | Mitigation |
| ---------------- | ---------- |
| SQL injection via sort param | Whitelist allowed column names |
| Negative page numbers | Clamp to minimum 1 |
| Huge limit values | Cap `limit` at reasonable max |

# 8. Tabs with URL State

Sync tab selection to the URL so users can share a specific tab.

```tsx
'use client'

import Link from 'next/link'
import { useSearchParams } from 'next/navigation'
import { Suspense } from 'react'

const TABS = ['overview', 'reviews', 'specs'] as const
type Tab = (typeof TABS)[number]

interface ProductTabsProps {
  productId: string
}

function ProductTabsInner({ productId }: ProductTabsProps) {
  const searchParams = useSearchParams()
  const activeTab = (searchParams.get('tab') as Tab) ?? 'overview'

  return (
    <div>
      <nav aria-label="Product tabs" role="tablist">
        {TABS.map((tab) => (
          <Link
            key={tab}
            href={`/products/${productId}?tab=${tab}`}
            role="tab"
            aria-selected={activeTab === tab}
          >
            {tab.charAt(0).toUpperCase() + tab.slice(1)}
          </Link>
        ))}
      </nav>

      <div role="tabpanel">
        {activeTab === 'overview' && <p>Product overview content</p>}
        {activeTab === 'reviews' && <p>Customer reviews</p>}
        {activeTab === 'specs' && <p>Technical specifications</p>}
      </div>
    </div>
  )
}

export function ProductTabs(props: ProductTabsProps) {
  return (
    <Suspense fallback={<p>Loading tabs...</p>}>
      <ProductTabsInner {...props} />
    </Suspense>
  )
}
```

# 9. nuqs for URL State Management

The `nuqs` library simplifies typed search param state on the client.

```tsx
'use client'

import { useQueryState, parseAsInteger, parseAsString } from 'nuqs'

export function ProductToolbar() {
  const [category, setCategory] = useQueryState('category', parseAsString.withDefault(''))
  const [sort, setSort] = useQueryState('sort', parseAsString.withDefault('name'))
  const [page, setPage] = useQueryState('page', parseAsInteger.withDefault(1))

  function handleCategoryChange(newCategory: string) {
    setCategory(newCategory || null) // null removes param from URL
    setPage(1) // reset page when filter changes
  }

  return (
    <div>
      <select
        value={category}
        onChange={(e) => handleCategoryChange(e.target.value)}
        aria-label="Filter by category"
      >
        <option value="">All categories</option>
        <option value="electronics">Electronics</option>
        <option value="clothing">Clothing</option>
      </select>

      <select
        value={sort}
        onChange={(e) => setSort(e.target.value)}
        aria-label="Sort products"
      >
        <option value="name">Name</option>
        <option value="price">Price</option>
      </select>

      <p>Page: {page}</p>
    </div>
  )
}
```

| Raw `useSearchParams` | `nuqs` |
| --------------------- | ------ |
| Manual string parsing | Typed parsers (`parseAsInteger`, etc.) |
| Manual URL building | Automatic sync with `setState`-like API |
| No default values built-in | `.withDefault()` helper |

Install: `npm install nuqs`

# 10. Server vs Client Strategy

| Use server `searchParams` | Use client `useSearchParams` / `nuqs` |
| ------------------------- | ------------------------------------- |
| Initial page render with filters | Live filter updates without full reload |
| SEO-critical listing pages | Complex multi-control filter panels |
| Zero JS filter links | Debounced search input |
| Pagination with `<Link>` | Syncing modal state to URL |

**Best pattern:** Server reads params for initial data fetch; client components update URL for interactivity.

```tsx
// app/products/page.tsx â€” Server reads params, fetches data
export default async function ProductsPage({ searchParams }: ProductsPageProps) {
  const params = await searchParams
  const products = await getProducts(params)

  return (
    <>
      <ProductFiltersClient initialParams={params} />
      <ProductList products={products} />
    </>
  )
}
```

# 11. Best Practices

âś” Prefer URL state over `useState` for filters, sort, pagination, and tabs

âś” Always `await searchParams` in server components (Next.js 15+)

âś” Validate and whitelist param values before using in queries

âś” Reset `page` to 1 when filters or search query change

âś” Wrap `useSearchParams` components in `<Suspense>`

âś” Use `<Link>` for filter links â€” works without JavaScript

âś” Consider `nuqs` for complex client-side URL state
