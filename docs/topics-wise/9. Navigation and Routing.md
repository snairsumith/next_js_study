# Navigation and Routing - Teaching Notes

## Topic 9: Link, Programmatic Navigation, and Active States

---

# 1. Why This Topic Matters

Routing is how users move through an app. Next.js provides **file-based routes** plus navigation tools that enable fast, client-side transitions without full page reloads.

Students must know:

* When to use `<Link>` vs `<a>` vs programmatic navigation
* How server redirects differ from client routing
* How prefetching makes navigation feel instant
* How to highlight the active route

Poor navigation causes full page reloads, lost state, and broken back-button behavior.

---

# 2. File-Based Routing Recap

URLs come from the `app/` folder structure:

```text
app/
├── page.tsx              → /
├── about/page.tsx        → /about
├── products/
│   ├── page.tsx          → /products
│   └── [id]/page.tsx     → /products/123
└── blog/
    └── [...slug]/page.tsx → /blog/2026/nextjs-guide
```

Navigation tools connect these routes in the UI. No React Router setup required — unlike react-project-study:

```tsx
// react-project-study — manual route config
<Route path="/products" element={<Product />} />
<Route path="/login" element={<Login />} />
```

In Next.js, creating the file **is** creating the route.

# 3. Link Component for Client-Side Navigation

Import **`Link`** from `next/link` for internal navigation.

```tsx
// components/layout/Nav.tsx
import Link from 'next/link'

export function Nav() {
  return (
    <nav aria-label="Main navigation">
      <ul className="flex gap-4">
        <li>
          <Link href="/">Home</Link>
        </li>
        <li>
          <Link href="/products">Products</Link>
        </li>
        <li>
          <Link href="/about">About</Link>
        </li>
      </ul>
    </nav>
  )
}
```

| Feature | Detail |
| ------- | ------ |
| Client-side transition | No full page reload |
| Prefetching | Linked pages prefetched in viewport by default |
| Replaces `<a>` for internal links | Use `<a>` only for external URLs |
| Works in Server Components | ✅ `Link` does not require `'use client'` |

### Link vs anchor tag

| Tag | Use for |
| --- | ------- |
| `<Link href="/about">` | Internal app routes |
| `<a href="https://example.com">` | External websites |
| `<a href="/file.pdf" download>` | File downloads |
| `<a href="mailto:support@example.com">` | Email links |

```tsx
// ❌ Wrong — full page reload for internal route
<a href="/products">Products</a>

// ✅ Correct — client-side navigation
<Link href="/products">Products</Link>
```

### Link with dynamic href

```tsx
<Link href={`/products/${product.id}`}>
  {product.name}
</Link>
```

```tsx
// With query params
<Link href="/products?category=electronics&sort=price">
  Electronics
</Link>
```

# 4. Link Props and Behavior

```tsx
<Link
  href="/products"
  prefetch={true}       // default — prefetch when in viewport
  replace={false}       // false = push history entry; true = replace
  scroll={true}         // scroll to top on navigation
  className="nav-link"
>
  Products
</Link>
```

| Prop | Default | Purpose |
| ---- | ------- | ------- |
| `href` | required | Destination path or URL object |
| `prefetch` | `true` | Preload route in background |
| `replace` | `false` | Replace history instead of push |
| `scroll` | `true` | Scroll to top after navigation |
| `passHref` | — | Legacy — rarely needed in App Router |

### Disable prefetch for heavy routes

```tsx
<Link href="/admin/heavy-report" prefetch={false}>
  Heavy Report (prefetch off)
</Link>
```

Use when the linked page is rarely visited or very expensive to prefetch.

# 5. useRouter and usePathname (Client)

For **programmatic navigation** and **reading the current path**, use client hooks from `next/navigation`.

```tsx
// components/SearchForm.tsx
'use client'

import { useRouter } from 'next/navigation'
import { useState, FormEvent } from 'react'

export function SearchForm() {
  const router = useRouter()
  const [query, setQuery] = useState('')

  function handleSubmit(event: FormEvent) {
    event.preventDefault()
    if (!query.trim()) return
    router.push(`/search?q=${encodeURIComponent(query)}`)
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="search"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search products..."
        aria-label="Search"
      />
      <button type="submit">Search</button>
    </form>
  )
}
```

```tsx
// components/layout/NavLink.tsx
'use client'

import Link from 'next/link'
import { usePathname } from 'next/navigation'
import clsx from 'clsx'

interface NavLinkProps {
  href: string
  label: string
}

export function NavLink({ href, label }: NavLinkProps) {
  const pathname = usePathname()
  const isActive = pathname === href

  return (
    <Link
      href={href}
      className={clsx(
        'px-3 py-2 rounded',
        isActive
          ? 'bg-brand-500 text-white font-semibold'
          : 'text-gray-600 hover:bg-gray-100',
      )}
      aria-current={isActive ? 'page' : undefined}
    >
      {label}
    </Link>
  )
}
```

| Hook | Import | Purpose |
| ---- | ------ | ------- |
| `useRouter()` | `next/navigation` | Programmatic push, replace, refresh, back |
| `usePathname()` | `next/navigation` | Current pathname string |
| `useSearchParams()` | `next/navigation` | Current URL query params (client) |

### useRouter methods

```tsx
'use client'

import { useRouter } from 'next/navigation'

export function ActionButtons() {
  const router = useRouter()

  return (
    <div>
      <button onClick={() => router.push('/dashboard')}>Go to Dashboard</button>
      <button onClick={() => router.replace('/login')}>Replace with Login</button>
      <button onClick={() => router.back()}>Go Back</button>
      <button onClick={() => router.refresh()}>Refresh Server Data</button>
    </div>
  )
}
```

| Method | Behavior |
| ------ | -------- |
| `router.push(href)` | Navigate forward — adds history entry |
| `router.replace(href)` | Navigate without adding history entry |
| `router.back()` | Browser back button equivalent |
| `router.forward()` | Browser forward button equivalent |
| `router.refresh()` | Re-fetch Server Components for current route |

**Important:** Use `next/navigation` in App Router — NOT `next/router` (Pages Router legacy).

Compare with react-project-study:

```tsx
// react-project-study — React Router
import { useNavigate, useLocation } from 'react-router-dom'

const navigate = useNavigate()
navigate('/products')
```

Next.js App Router uses `useRouter` from `next/navigation` instead.

# 6. redirect() and permanentRedirect() (Server)

For navigation triggered on the **server** — auth guards, form processing, missing data — use **`redirect()`** from `next/navigation`.

```tsx
// app/dashboard/page.tsx
import { redirect } from 'next/navigation'
import { getSession } from '@/lib/auth'

export default async function DashboardPage() {
  const session = await getSession()

  if (!session) {
    redirect('/login')
  }

  return <h1>Welcome, {session.user.name}</h1>
}
```

```tsx
// permanentRedirect — HTTP 308 (SEO: old URL permanently moved)
import { permanentRedirect } from 'next/navigation'

export default async function OldBlogPage() {
  permanentRedirect('/blog/new-location')
}
```

| Function | HTTP status | Use case |
| -------- | ----------- | -------- |
| `redirect(url)` | 307 (temporary) | Auth checks, post-action navigation |
| `permanentRedirect(url)` | 308 (permanent) | URL migrations, SEO redirects |

### Server vs client navigation

| Trigger | Tool |
| ------- | ---- |
| User clicks link | `<Link>` |
| User submits client form | `router.push()` |
| Server detects unauthorized | `redirect()` |
| Server Action after mutation | `redirect()` or `revalidatePath()` |
| Button click in client component | `router.push()` |

```tsx
// ❌ redirect() inside Client Component — wrong
'use client'
import { redirect } from 'next/navigation' // throws — use router.push instead

// ✅ redirect() in Server Component or Server Action
export default async function Page() {
  redirect('/home')
}
```

# 7. Active Link Styling

Highlight the current route in navigation for better UX.

### Basic exact match

```tsx
'use client'

import Link from 'next/link'
import { usePathname } from 'next/navigation'

export function MainNav() {
  const pathname = usePathname()

  const links = [
    { href: '/', label: 'Home' },
    { href: '/products', label: 'Products' },
    { href: '/about', label: 'About' },
  ]

  return (
    <nav>
      {links.map(({ href, label }) => {
        const isActive = pathname === href
        return (
          <Link
            key={href}
            href={href}
            aria-current={isActive ? 'page' : undefined}
            className={isActive ? 'font-bold underline' : ''}
          >
            {label}
          </Link>
        )
      })}
    </nav>
  )
}
```

### Prefix match for nested routes

```tsx
const isActive = pathname === href || pathname.startsWith(`${href}/`)

// /products active when on /products or /products/123
const isProductsActive = pathname.startsWith('/products')
```

| Match type | Logic | Example |
| ---------- | ----- | ------- |
| Exact | `pathname === href` | `/about` active only on `/about` |
| Prefix | `pathname.startsWith(href)` | `/products` active on `/products/42` |
| Segment | Split pathname and compare | More control for complex nav |

Always add `aria-current="page"` on the active link for accessibility.

# 8. Programmatic Navigation Patterns

### After form validation (client)

```tsx
'use client'

import { useRouter } from 'next/navigation'

export function LoginForm() {
  const router = useRouter()

  async function handleLogin(formData: FormData) {
    const email = formData.get('email') as string
    const password = formData.get('password') as string

    const res = await fetch('/api/login', {
      method: 'POST',
      body: JSON.stringify({ email, password }),
    })

    if (res.ok) {
      router.push('/dashboard')
    }
  }

  return (
    <form action={handleLogin}>
      <input name="email" type="email" required />
      <input name="password" type="password" required />
      <button type="submit">Login</button>
    </form>
  )
}
```

### Conditional navigation

```tsx
function handleNextStep(step: number) {
  if (step === 1) router.push('/checkout/shipping')
  if (step === 2) router.push('/checkout/payment')
  if (step === 3) router.push('/checkout/confirm')
}
```

### Refresh after mutation

```tsx
async function handleDelete(id: string) {
  await deleteItem(id)
  router.refresh() // re-run Server Components without full navigation
}
```

# 9. Back and Forward Behavior

Next.js App Router integrates with the **browser history API**.

| Action | Behavior |
| ------ | -------- |
| `Link` click | Pushes new entry — back button returns |
| `router.push()` | Same as Link — adds history |
| `router.replace()` | Replaces current entry — back skips replaced page |
| `router.back()` | Same as browser back button |
| Layout persistence | Layouts stay mounted when using back/forward within same layout segment |

```tsx
'use client'

import { useRouter } from 'next/navigation'

export function BackButton() {
  const router = useRouter()

  return (
    <button onClick={() => router.back()} aria-label="Go back">
      ← Back
    </button>
  )
}
```

### When to use replace vs push

| Scenario | Method |
| -------- | ------ |
| Normal navigation | `push` / `<Link>` |
| Redirect after login (don't return to login on back) | `replace` |
| Redirect after logout | `replace` |
| Wizard step correction | `replace` optional |

# 10. Prefetching with Link

Next.js **prefetches** linked routes when `<Link>` enters the viewport — making navigation feel instant.

```text
User sees Link to /products
       │
       ▼
Next.js prefetches /products JS and RSC payload in background
       │
       ▼
User clicks Link → near-instant navigation
```

| Prefetch setting | Behavior |
| ---------------- | -------- |
| `prefetch={true}` (default) | Prefetch when link visible |
| `prefetch={false}` | No prefetch — navigate on click only |
| Production only | Prefetch disabled in development for some routes |

### Optimize prefetch strategy

```tsx
{/* High-traffic route — keep default prefetch */}
<Link href="/products">Products</Link>

{/* Rarely visited heavy admin page — disable */}
<Link href="/admin/export" prefetch={false}>Export</Link>
```

Prefetch works with static and dynamic routes. Dynamic routes prefetch up to the loading boundary.

# 11. Complete Navigation Example

```tsx
// components/layout/SiteHeader.tsx — Server Component shell
import Link from 'next/link'
import { NavLinks } from './NavLinks'
import { SearchForm } from './SearchForm'

export function SiteHeader() {
  return (
    <header className="border-b">
      <div className="max-w-6xl mx-auto flex items-center justify-between p-4">
        <Link href="/" className="text-xl font-bold">
          Training Shop
        </Link>
        <NavLinks />
        <SearchForm />
      </div>
    </header>
  )
}
```

```tsx
// components/layout/NavLinks.tsx
'use client'

import Link from 'next/link'
import { usePathname } from 'next/navigation'
import clsx from 'clsx'

const NAV_ITEMS = [
  { href: '/', label: 'Home', exact: true },
  { href: '/products', label: 'Products', exact: false },
  { href: '/about', label: 'About', exact: true },
]

export function NavLinks() {
  const pathname = usePathname()

  return (
    <nav className="flex gap-2" aria-label="Main">
      {NAV_ITEMS.map(({ href, label, exact }) => {
        const isActive = exact
          ? pathname === href
          : pathname.startsWith(href)

        return (
          <Link
            key={href}
            href={href}
            aria-current={isActive ? 'page' : undefined}
            className={clsx(
              'px-3 py-1 rounded',
              isActive ? 'bg-brand-500 text-white' : 'hover:bg-gray-100',
            )}
          >
            {label}
          </Link>
        )
      })}
    </nav>
  )
}
```

# 13. Best Practices

✔ Use **`<Link>`** for all internal navigation — never raw `<a href="/...">`

✔ Use **`redirect()`** on the server for auth and conditional routing

✔ Use **`useRouter().push()`** for client-side programmatic navigation after user actions

✔ Import from **`next/navigation`** — not `next/router`

✔ Add **`aria-current="page"`** on active nav links

✔ Split nav into **Server shell + Client active-state** component when using `usePathname`

✔ Use **`router.replace()`** after login/logout to prevent back-button loops

✔ Disable **`prefetch={false}`** only for heavy or rarely visited routes

✔ **`encodeURIComponent`** when building query strings manually

# 15. Summary

| Tool | When to use |
| ---- | ----------- |
| `<Link href="">` | Internal links — default choice |
| `<a href="">` | External URLs only |
| `useRouter().push()` | Client programmatic navigation |
| `useRouter().replace()` | Replace history (login/logout) |
| `useRouter().back()` | Back button behavior |
| `useRouter().refresh()` | Re-fetch server data on current route |
| `usePathname()` | Active link styling |
| `redirect()` | Server-side redirect (auth, guards) |
| `prefetch` | Default on Link; disable for heavy routes |
| vs React Router | `href` not `to`; `next/navigation` not `react-router-dom` |
